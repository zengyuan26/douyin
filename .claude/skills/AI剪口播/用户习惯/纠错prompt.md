# ASR 纠错规则

## 输入输出

输入：ASR 转录粗稿（每句一行）
输出：纠错后的文本（每句一行，与输入 1:1 对应，不做断行、不去标点）

## 核心前提

**输入对应一条剪好的成片视频**。逻辑一定通顺、顺序一定正确、严重口误已被剪掉。只保留自然的说话习惯（语气词、口头禅）。

**核心判断问：这句话在成片里，说话人真的会这么说吗？**
- 不会 → 是 ASR 错误，必须改
- 会，只是口头禅/语气词 → 保留

## 允许的四种操作

| 操作 | 场景 |
|---|---|
| 替换 | 同音字、谐音、英文误识别、拼音乱码 |
| 补漏字 | ASR 漏识别的单字 |
| 去重复 | ASR 叠字 |
| 保留 | 确实拿不准 → 保留原样，完成后列「不确定清单」给用户 |

## 必须保留

说话者的自然口头禅：呢、吧、就、直接、其实、然后、这个、那个、的话。这些是成片里正常存在的。

## 必须修正（ASR 错误信号）

- 逻辑不通、语义断裂
- 顺序混乱、语法错乱
- 明显重复（ASR 叠字）
- 单字+功能描述（多是英文词被 ASR 错译）
- 拼音乱码（英文被错转成拼音串）

## 专有名词

- 产品型号、品牌名：结合上下文/视频主题推断
- 模型名：不确定时**保留原 ASR 词**，不要瞎猜 Sonnet/Opus/Haiku

## 本步骤不做

- 不断行
- 不去标点
- 不改大小写
- 不加空格调整
