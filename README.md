# 🌍 Geo Master - 多平台运营总控系统

## 📚 项目概述

这是一个**多平台运营Skill系统**，采用Master-Worker架构设计。系统包含**8个核心Skill**（客户资料收集已整合到总控），覆盖从市场洞察、社交舆情监控、SEO优化、内容创作、视觉设计、消费心理学、运营规划到总控调度的全链路能力。

适用于**抖音、小红书、快手、微信视频号**等各类内容平台。

## 💡 核心思维：问题在先，人群在后

### ❌ 错误思路（先有人，再找需求）

```
先定义人群：新手妈妈、二胎妈妈
↓ 
找需求：担心假货、不会选
↓ 
卖产品：进口奶粉
```

### ✅ 正确思路（先有问题，再有人群）

```
先发现问题：孩子乳糖不耐受、便秘、过敏
↓ 
人群自动出现：有个这样的宝宝的妈妈
↓ 
提供解决方案：特殊奶粉 + 专业指导
```

### 核心逻辑

```
问题 → 背后站的是人群 → 需要解决方案

孩子有问题 → 妈妈才需要买奶粉
水管漏水 → 业主才需要修水管
```

### 为什么这样是蓝海？

| 思路 | 竞争 | 结果 |
|------|------|------|
| ❌ 先人群：新手妈妈 | 红海，大家都在抢 | 难出头 |
| ✅ 先问题：乳糖不耐受 | 蓝海，问题导向 | 精准流量 |

**中国人口基数大，再小的需求也有很多人！**

## 💡 进阶思维：产品服务化

> 💡 **核心理念**：卖产品不如卖服务，产品只是服务的载体

### ❌ 传统思路（单纯卖产品）

```
定位：我是卖XX产品的
思维：产品思维、交易思维
结果：陷入价格战、利润越来越薄
```

### ✅ 服务化思路（卖解决方案）

```
定位：我是提供XX解决方案/服务的
思维：服务思维、顾问思维、医生思维
结果：建立专业壁垒、难以被替代、定价权
```

### 产品服务化的三种形式

| 形式 | 说明 | 示例 |
|------|------|------|
| **顾问型** | 成为客户的咨询顾问 | 卖锁 → 安防顾问/开锁专家 |
| **医生型** | 像医生一样诊断+方案 | 卖电池 → 电池医生/电瓶诊疗 |
| **服务型** | 产品是服务的载体 | 卖水 → 企业饮水解决方案 |

### 应用案例

```
案例1：卖锁
- ❌ 定位：卖锁的
- ✅ 定位：家庭安防顾问/开锁专家

案例2：卖电池
- ❌ 定位：卖电池的
- ✅ 定位：电池医生/电瓶养护专家

案例3：卖水
- ❌ 定位：卖定制水的
- ✅ 定位：企业饮水解决方案专家
```

### 在项目中的应用

| Skill | 应用场景 |
|-------|----------|
| 运营专家 | 帮客户重新定位业务，打造专家/顾问人设 |
| Geo SEO | 关键词布局体现服务价值，内容围绕问题解决 |
| 内容创作 | 创作内容时体现服务价值而非产品功能 |
| 行业分析 | 分析客户时帮助重新定义业务为服务 |
| 客户收集 | 收集信息时引导客户重新定义业务 |
| 互联网营销 | 昵称和简介体现服务/专家身份 |

## 📞 专家呼出命令

**直接输入以下命令即可呼出对应专家：**

| 命令 | 专家 | 功能 |
|------|------|------|
| `/总控` / `/master` | Geo Master | 开始新客户流程（清除当前客户） |
| `/切换客户` | 切换客户 | 列出已有客户，切换当前客户 |
| `/新增客户` | 新增客户 | 引导新增客户并设为当前客户（整合客户资料收集） |
| `/seo` | Geo SEO专家 | 关键词挖掘、选题规划、SEO优化 |
| `/运营` | 运营专家 | 市场机会发现、差异化定位、账号设计、运营方案 |
| `/市场` | 市场洞察专家 | 行业分析、竞品研究、趋势监控 |
| `/舆情` | 抖音舆情监控专家 | 实时监控、舆情分析、竞品追踪、风险预警（仅当前客户） |
| `/内容` | 内容专家 | 图文规划、短视频脚本、封面设计 |
| `/创作` | 内容创作者 | 具体脚本创作、图文内容生成 |
| `/心理` | 消费心理专家 | 消费心理分析、文案优化、转化提升 |
| `/视觉` | 视觉设计专家 | 视觉设计评审、9:16比例检查、场景图适配 |

**使用方式**：
- 输入命令如 `/seo` 即可呼出对应专家
- 专家被呼出后会自动展示个人介绍和能力清单
- 除非明确呼叫总控（`/总控`），否则默认使用当前客户
- 客户资料收集已整合到总控（`/新增客户`、`/切换客户`）

### 抖音舆情监控专家（`/舆情`）特别说明
- **只监控抖音平台**：不再泛指"全网社交媒体"
- **只服务当前客户**：每次调用自动读取 `系统配置/当前客户.json`，显示当前客户名称
- **每个客户最多10个监控链接**：包含话题、视频、商品三种类型
- **定时任务**：每天早上6:00自动执行，生成数据报告到 `舆情专家/数据报告/`

## 🎯 核心功能

| 收集维度 | 说明 |

| 客户类型 | 卖货类 / 服务类 / 两者都有 |

**📊 关键词配比原则（起步阶段0-30天）**：

```
✅ 蓝海长尾词：50%+ → 精准流量，竞争小，容易做起来
✅ 问题词/场景词：35% → 真实需求，转化率高
⚠️ 红海大词：≤15% → 起步阶段不做或极少量，做账号权重
🎯 整体思路：切细分蓝海市场，小词拉大词，做账号权重
```

## 🚀 快速开始

### 1. 用户首次咨询

```
用户：我是卖奶粉的，帮我做运营

→ Geo Master 启动 → 自我介绍
→ 引导用户逐项填写客户信息
→ 生成客户档案
→ 调用 Operations Expert 深度调研
→ 调用 Insights Analyst 行业分析
→ 调用 Geo SEO 关键词库+选题库
→ 确定专家组合，启动内容生成
```

### 2. 完整工作流程

```
Step 1: 用户启动（说"开始答题"或新客户）
   ↓
Step 2: Geo Master 自我介绍 + 引导问卷
   ↓
Step 3: 创建客户档案（运行新增客户.py）
   ↓
Step 4: 收集客户基础信息（Client Intake）
   ↓
Step 5: 深度运营调研（Operations Expert）
   ↓
Step 6: 行业分析报告（Insights Analyst）
   ↓
Step 7: 关键词库 + 选题库（Geo SEO）
   ↓
Step 8: 内容生成（Geo SEO + Content Expert）
   ↓
Step 9: 心理学审核（Consumer Psychology Expert）
   ↓
Step 10: 整合输出完整方案
```

### 3. 专家调度规则

```
必选专家 = Geo SEO（所有项目）

必选调度 =
  - Operations Expert（新客户首合作）
  - Consumer Psychology Expert ⭐（所有内容审核）

可选调度 =
  - 若需市场洞察 → 调度 Insights Analyst
  - 若需内容创作 → 调度 Content Expert
```

## 📋 客户管理

### 客户档案结构

```
客户管理/
└── [行业]/
    └── [业务范围]/
        └── [客户名称]/
            ├── 客户档案.json           # 客户信息
            ├── 品牌资料/               # logo、品牌故事
            ├── 产品资料/               # 产品图片、参数
            ├── 包装设计/               # 包装素材
            ├── 人物形象/               # 出镜人物素材
            ├── 场景素材/               # 拍摄场景
            ├── 宣传物料/               # 宣传素材
            └── 参考案例/               # 参考案例
```

### 客户档案字段

| 字段 | 说明 |
|------|------|

| client_name | 客户名称 |
