> 基于 Flask + SQLite 的 Web 应用系统
> 系统简称：AI棱镜
> 版本：v3.1.9
> 更新日期：2026-03-05

---

## 一、系统概述

### 1.1 产品定位

**AI棱镜** 是一套基于 Master-Worker 架构的**多平台内容营销智能系统**。采用"对话+工具"的混合模式，让渠道用户和超级管理员能够通过类似 ChatGPT 的对话界面，与本地 AI 专家团队高效协作，完成客户资料收集、市场分析、SEO 优化、内容创作等全链路营销工作。

### 1.2 核心特色

- **本地大模型驱动**：基于 Ollama 本地部署，支持 Qwen、Llama、DeepSeek 等模型
- **对话式交互**：像聊天一样使用 AI 专家服务
- **智能调度**：首席营销官自动分配任务给合适的专家
- **一站式管理**：客户、关键词、选题、内容、监控全覆盖
- **多平台支持**：抖音、小红书、快手、视频号等

### 1.3 本地大模型配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| LLM_PROVIDER | ollama | 模型提供商 (ollama/openai/azure) |
| LLM_MODEL | qwen2.5:7b | 模型名称 |
| LLM_BASE_URL | http://localhost:11434 | API 地址 |
| LLM_API_KEY | (空) | API 密钥（OpenAI/Azure 需要）|

**支持的本地模型**：
- 通义千问 (Qwen2.5)
- Llama 3/4
- DeepSeek
- Mixtral
- Phi

**快速测试**：
```bash
# 启动 Ollama
ollama serve

# 测试模型
ollama run qwen2.5:7b "你好"

# 或通过 API 测试
curl http://localhost:11434/api/tags
```

## 三、开发规则（基于 .cursorrules）

### 3.1 设计原则

**必须遵循 Apple 设计美学**：
- 精致的字体系统、半透明效果、细节光影、材质系统
- 严格统一的排版系统
- 绝对禁止使用 emoji 表情
- 所有设计元素必须保持高度一致性

### 3.2 视觉层级

- 使用大小、重量、色彩、对比度和空间位置创建清晰的视觉层级
- 最重要的信息必须视觉上最突出
- 保持 5-6 个清晰区分的层级
- 所有文本必须严格左对齐

### 3.3 空间与留白

- 标题距离卡片顶边框 40px
- 页脚元素距离卡片底边框 40px
- 严格遵循一致的间距系统
- 右上角预留二维码放置区域

### 3.4 质量检查

1. 所有元素严格对齐，无偏移或不对称
2. 序号/列表符号系统统一，无双重序号符号
3. 文本格式统一，无随机字体或大小变化
4. 精简后的内容保留原文核心逻辑和关键点
5. **所有弹框一律居中显示**，使用 Bootstrap 的 `modal-dialog-centered` 类，CSS 中 `.modal-dialog` 必须使用 `margin: 1.75rem auto` 而非 `margin: var(--spacing-lg)`

## 四、用户流程

### 4.1 渠道用户首次流程使用

```
┌─────────────────────────────────────────────────────────────┐
│  渠道用户登录                                                 │
│       ↓                                                     │
│  进入工作台（默认只有「首席营销官」专家）                       │
│       ↓                                                     │
│  点击「+」新增客户 或 输入「@新增客户」                        │
│       ↓                                                     │
│  首席营销官 发起对话式问卷收集客户资料                        │
│       ↓                                                     │
│  用户逐项回答问题（每次只问一个问题）                          │
│       ↓                                                     │
│  首席营销官 生成「客户档案报告」                              │
│       ↓                                                     │
│  自动分配给其他专家：                                         │
│    • 市场洞察分析师 → 行业分析报告                            │
│    • 社交舆情监控专家 → 监控配置                             │
│    • Geo SEO 专家 → 关键词库+选题库                          │
│    • 运营专家 → 运营规划方案                                 │
│    • 内容专家 → 内容脚本/图文规划                             │
│       ↓                                                     │
│  用户可在左侧专家列表切换查看各专家输出                        │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 专家列表展示顺序

```
1. 首席营销官（总控）- 默认显示
2. 市场洞察分析师
3. 社交舆情监控专家
4. Geo SEO 专家
5. 运营专家
6. 内容专家
7. 消费心理学专家
8. 视觉设计专家
```

## 五、功能模块

### 4.1 首席营销官（总控专家）

**职责**：需求分析、客户资料收集、专家调度、方案整合

**对话式收集维度**：
| 维度 | 说明 | 示例 |
|------|------|------|
| 客户类型 | 卖货类/服务类/两者都有 | 卖货类 |
| 地域范围 | 本地/跨区域/全球/本地+跨区域 | 本地+跨区域 |
| 品牌定位 | 个人IP/公司品牌/两者兼顾 | 个人IP |
| 语言风格 | 普通话/方言 | 四川话 |
| 产品/服务信息 | 主营业务 + 其他补充 | 婴幼儿奶粉 |

**输出**：
- 客户档案报告
- 自动调度其他专家
- 方案整合

**命令**：
- `/总控` 或 `/master` - 重新召唤首席营销官
- `/新增客户` - 开始新客户资料收集
- `/切换客户` - 切换到已有客户

### 4.2 市场洞察分析师

**职责**：行业分析、竞品研究、趋势监控

**功能模块**：
- 行业分析：市场规模、增长率、竞争格局
- 竞品分析：头部竞品内容策略、互动数据
- 需求挖掘：用户需求分布、增长最快的需求
- 趋势预测：季节性热点、即将爆发的选题

**输出**：
- 行业分析报告 → `输出/行业分析/{客户名}/`
- 竞品分析报告
- 趋势预测报告

### 4.3 社交舆情监控专家

**职责**：抖音平台舆情监控

**功能模块**：
- 实时监控：话题、视频、商品监控
- 舆情分析：正面/中性/负面分析
- 竞品追踪：竞品动态监控
- 风险预警：负面舆情告警

**配置限制**：每个客户最多 10 个监控链接

**输出**：
- 监控配置 → `舆情专家/{客户名}/监控配置.md`
- 数据报告 → `舆情专家/数据报告/`（每日自动生成）

### 4.4 Geo SEO 专家

**职责**：关键词挖掘、选题规划、SEO 优化

**功能模块**：
- 关键词矩阵：核心词/产品词/长尾词/地域词
- 蓝海挖掘：大词检测、差异化长尾词
- 选题规划：多场景选题库、选题变体矩阵
- 内容生成：6种内容结构模板
- 标题优化：5种标题类型
- 标签矩阵：Hashtag 公式
- 方言适配：四川话/粤语/东北话
- 首评引导：5种首评类型

**关键词配比原则（起步阶段 0-30 天）**：
```
蓝海长尾词：50%+ → 精准流量
问题词/场景词：35% → 转化率高
红海大词：≤15% → 做账号权重
```

**输出**：
- 关键词库 → `输出/关键词库/{客户名}/`
- 选题推荐 → `输出/选题推荐/{客户名}/`

### 4.5 运营专家

**职责**：市场机会发现、差异化定位、账号设计、运营规划

**功能模块**：
- 深度调研：行业经验、竞争优势、资源情况
- 账号设计：昵称、头像、简介、背景图
- 竞争分析：竞品分析、差异化定位
- 运营规划：人设定位、内容矩阵、变现路径
- 信任构建：本地vs跨区域、个人IPvs品牌

**输出**：
- 运营规划方案 → `输出/运营规划/{客户名}/`

### 4.6 内容专家

**职责**：图文规划、短视频脚本、封面设计

**功能模块**：
- 脚本撰写：6种脚本结构模板
- 封面设计：6种封面类型模板
- 互动引导：评论区引导话术库
- 模板库：通用脚本模板、封面模板
- 场景选择：同一选题多个场景供选择

**输出**：
- 内容脚本 → `输出/内容脚本/{客户名}/`
- 图文规划 → `输出/图文规划/{客户名}/`

### 4.7 消费心理学专家

**职责**：心理学角度优化内容

**功能模块**：
- 心理钩子：3种钩子类型（打断/奖励/损失）
- 信任构建：身份信任/经验信任/逻辑信任
- 说服力：AIDA模型、5种说服工具
- 转化路径：认知→兴趣→信任→欲望→行动
- 方言心理学：方言类型与心理适配
- 语音风格：6种语音风格心理学

**使用方式**：作为其他专家的"审核者"角色，对生成内容进行心理学审核

### 4.8 视觉设计专家

**职责**：视觉设计评审、9:16 比例检查

**功能模块**：
- 视觉设计评审
- 9:16 比例检查
- 场景图适配
- 排版优化

**使用方式**：作为内容专家的"审核者"角色

## 十一、专家 Skill 系统

### 11.1 专家配置文件结构

每个专家对应一个 Skill 目录，包含：

```
skills/
├── geo-master/
│   ├── skill.md           # 主配置文件
│   ├── prompt.md          # 系统提示词
│   └── examples/          # 示例对话
```

### 11.2 专家调用流程

```
用户输入 → 判断命令/意图
    ↓
路由到对应专家
    ↓
加载专家 Skill 配置
    ↓
构建 Prompt（历史对话 + Skill + 客户信息）
    ↓
调用 LLM 服务
    ↓
返回结果并存储
```

### 11.3 Skill 配置字段

| 字段 | 说明 |
|------|------|
| name | 专家名称 |
| slug | 唯一标识 |
| title | 职位/头衔 |
| description | 描述 |
| capabilities | 能力列表 |
| system_prompt | 系统提示词 |
| command | 呼出命令 |

### 11.4 按需加载机制

系统采用按需加载设计，只有用户调用某个专家时，才加载对应的 skill.md 文件：

- **优势**：
  - 减少内存占用
  - 支持动态更新 skill 内容
  - 首次加载速度更快

- **实现**：
  - `SkillLoader` 类负责加载和管理 skills
  - 使用缓存机制避免重复读取文件
  - 支持 `clear_cache()` 手动刷新

### 11.5 安全权限控制

为保护系统安全，skill 修改权限进行了严格控制：

#### 11.5.1 用户角色

| 角色 | 权限 |
|------|------|
| super_admin | 可读取和使用 skill，可修改 skill 内容 |
| channel | 可读取和使用 skill，不可修改 |
| user | 可读取和使用 skill，不可修改 |

#### 11.5.2 安全机制

1. **输入检测**：在消息进入 LLM 前检测危险请求
2. **System Prompt 注入**：根据用户角色动态注入安全指令
3. **危险关键词拦截**：
   - 修改、更新、编辑、删除
   - 写入、保存、创建文件
   - 管理员、权限、后门、绕过

#### 11.5.3 拦截响应

普通用户请求修改 skill 时返回：
```
"Sorry, no permission. Contact admin."
```
