## 三、分析维度管理

### 设计思路

分析维度是**爆款拆解/方法论学习**时，AI 分析内容的「检查项」。你配置得越细，AI 分析得越深入。

**核心特点**：
- **即时生效**：新增维度后立即可用，无需重启
- **自由发挥**：AI在按维度分析的同时，额外给出分析维度的建议
- **公式入库**：AI分析后**总结出的公式**才是入规则库的东西（不是维度本身）

### 三个分类

| 分类 | 说明 | 例子 |
|------|------|------|
| **账号分析** | 分析账号相关的维度 | 昵称、头像、简介、定位、人设 |
| **内容分析** | 分析内容本身的维度 | 标题、封面、开头钩子、结尾、结构、选题 |
| **方法论** | 通用方法论/理论 | 消费心理学、用户行为模型、转化逻辑 |

### 每个维度的配置项

```
维度配置
├── 维度名称（如：标题结构、开头钩子、易记性分析）
├── 一级分类（账号/内容/方法论）
├── 二级分类（可选，更细分组）
├── 描述（可选，简短说明）
│
├── 规则库入库配置（可选）
│   ├── 入库到规则库的哪个分类
│   │   ├── 关键词库
│   │   ├── 选题库
│   │   ├── 内容模板
│   │   ├── 运营规划
│   │   └── 市场分析
│   │
│   └── 规则类型标识（英文唯一标识）
│       例如：account_design_nickname、content_title、content_hook
│       → 用于后续生成报告时按类型筛选
│
└── 排序 + 是否默认
```

### 核心流程：维度 → 分析 → 公式 → 入库

```
1. 配置分析维度（如：昵称、头像、简介...）
         │
         ▼
2. 爆款拆解时，LLM 按维度分析内容
         │
         ▼
3. LLM 在分析维度的基础上，总结出【公式】
   例如：
   - 维度：昵称
   - LLM 总结的公式：「昵称采用【名字+身份】组合，如XX妈、XX老师，
     能快速建立专业信任感」
         │
         ▼
4. 用户点「入库」→ 公式存入规则库对应分类
         │
         ▼
5. 规则库中显示的是「公式」，不是「维度」
   例如：
   - 账号分析 → 昵称 → 【公式1】、【公式2】...
```

**关于「分析提示词」的纠正**：
- 不是给 AI 的分析思路
- 而是让 AI **在维度基础上自由发挥**，额外给出分析维度的建议
- 相当于让 AI 帮你发现更多可能的分析角度

## 四、规则库

### 设计思路

规则库 = **可复用的知识资产**。所有从爆款拆解、方法论学习、规则自动提取入库的规则都放在这里，供生成报告时调用。

### 分类体系

```
规则库
├── 关键词库        # 关键词、流量词、转化词
├── 选题库         # 好的选题方向
├── 内容模板       # 标题模板、开头模板、结构模板
├── 运营规划       # 账号定位、人设、发布策略
└── 市场分析       # 行业趋势、竞品分析
```

### 每条规则的结构

```
规则
├── 标题（rule_title）
├── 内容（rule_content）
├── 分类（category）- 关键词库/选题库/内容/运营规划/市场分析
├── 类型标识（rule_type）- 来自哪个分析维度
│   例如：account_design_nickname、content_title
├── 规则类型（rule_type）- dimension/logic/structure/methodology
├── 适用场景（applicable_scenarios）- 种草/带货/品牌宣传
├── 适用人群（applicable_audiences）
├── 适用平台（platforms）- douyin/xhs/bilibili
└── 状态（status）- pending/active/archived
```

### 页面功能（简化版）

```
规则库页面
├── 列表展示（按分类分Tab）
├── 查看详情
└── 手动新增（可选）

不需要：搜索、筛选、编辑、删除、使用统计
既然是爆款拆解入库的，肯定是有价值的规则
```

## 五、两者的关联

```
分析维度（配置）                    规则库（数据）
┌─────────────────┐              ┌─────────────────┐
│ 维度A            │              │ 规则A1 (type:A) │
│ rule_category=X │  ──────────▶ │ 规则A2 (type:A) │
│ rule_type=a_xxx │              └─────────────────┘
└─────────────────┘              ┌─────────────────┐
┌─────────────────┐              │ 规则B1 (type:B) │
│ 维度B            │              └─────────────────┘
│ rule_category=Y │
│ rule_type=b_yyy │
└─────────────────┘
```

**核心流程**：
1. 配置分析维度（rule_category + rule_type）
2. 爆款拆解时自动加载所有启用的维度
3. AI按维度分析 + 额外给建议
4. 用户觉得好 → 点「入库」→ **先比对规则库是否有重复**
5. 比对结果：
   - 已有相似规则 → 提示"系统已有"，不强制入库
   - 没有重复 → 建议入库，用户判断是否入库
6. 入库成功 → 存到规则库对应分类
7. 生成报告时按 rule_type 精确匹配

## 六、确认后的需求

### 1. 方法论学习（待完善）

- **交互形式**：简化为一个「录入笔记 → 选择维度 → 分析 → 决定入库」的单页面
- **关键点**：书籍内容可能不涉及所有维度，所以让用户手动选择
- **定位**：另一种形式的爆款拆解，信息源从「短视频」变成「书本/笔记」

### 2. 规则自动提取（待完善）

- **触发时机**：自动触发（生成报告时）
- **入库流程**：审核后直接入库

### 3. 优先级

**第一阶段：优化爆款拆解**
- 确保自动加载维度生效
- AI额外给建议的功能
- 入库流程顺畅

**第二阶段：完善方法论学习**
- 新建单页面交互
- 支持手动选择分析维度

**第三阶段：规则自动提取**
- 生成报告时自动提取
- 完善审核入库流程

**第四阶段：清理不需要的功能**
- 删除使用统计
