# 目标客户画像：十维度与分批规则

> 与产品画像表一致；实现见 `services/public_content_generator.py`、`routes/public_api.py`（`/api/persona/dimensions`）。

## 十个维度（限定词示例 + 适用场景）

| 维度 | 限定词示例 | 适用场景 |
|------|------------|----------|
| **发展阶段** | 刚起步 / 天使轮 / A轮 / 成熟期 / 转型期 | 创业者、企业家 |
| **营收规模** | 500万以下 / 3000万-2亿 / 5亿以上 | 企业家、B2B服务 |
| **团队规模** | 10人以下 / 10-50人 / 50-400人 | 企业家、管理者 |
| **工作年限** | 3-5年 / 5-10年 / 10年以上 | 职场人、专家 |
| **行业背景** | 传统制造 / 互联网 / 金融 / 教育 | 所有人群 |
| **地域** | 一线城市 / 新一线 / 三四线 / 海外 | 本地服务、区域业务 |
| **年龄段** | 25-35岁 / 35-45岁 / 45岁以上 | 个人服务 |
| **职位角色** | 创始人 / CEO / VP / 总监 / 经理 | B2B服务 |
| **痛点状态** | 遇到瓶颈 / 想要转型 / 寻求突破 | 所有人群 |
| **目标诉求** | 想要增长 / 想要转型 / 想要变现 | 所有人群 |

## 分批与组合规则

1. 每次随机选 **3～4 个非「职位角色」维度**，再 **必选「职位角色」**。
2. **组合展示顺序**：先按上表顺序排列已选维度取值，**「职位角色」固定排在最后**（例：`成熟期 × 3000万-2亿 × 遇到瓶颈 × 想要增长 × 总监`）。
3. **同一批 5 条画像**：以本批选中的 **痛点状态、目标诉求** 等为共性锚点，保证痛点或目标一致；各条在职位角色等维度上区分人群。
