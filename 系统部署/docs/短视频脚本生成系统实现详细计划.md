## 二、API端点设计

### 2.1 评分API
```
POST /api/script/score
功能：对脚本进行多维度评分
请求体：
{
  "script": {
    "title": "...",
    "opening": "...",
    "scenes": [...],
    "narration": "..."
  },
  "trust_source": "knowledge|persona|institution|product"
}
响应：
{
  "success": true,
  "data": {
    "total_score": 85,
    "grade": "A(良好)",
    "passed": true,
    "emotion_score": 80,
    "rhythm_score": 85,
    "interaction_score": 90,
    "dimensions": [...],
    "suggestions": [...]
  }
}
```

### 2.2 业务分类API
```
POST /api/script/classify
功能：根据业务特征判断信任来源
请求体：
{
  "name": "客户名称",
  "industry": "行业",
  "org_type": "机构类型",
  "tags": ["标签1", "标签2"],
  "description": "描述"

响应：
{
  "success": true,
  "data": {
    "trust_source": "persona",

    "confidence": 0.85,
    "recommended_topic_types": ["人设价值观类", "人设故事类"],
    "recommended_balance_config": {...},
    "reasoning": "..."
  }
}
```

## 四、集成内容生成流程

### 4.1 在内容生成时调用评分
位置：`services/public_content_generator.py`
```python
# 生成内容后调用评分
if enable_scoring:
    from services.script_scorer import ScriptScorer
    scorer = ScriptScorer()
    report = scorer.score(script_data, trust_source_type)
    result["quality_report"] = report.to_dict()
```

### 4.2 在内容生成时计算奖励点
```python
# 生成内容后计算奖励点分布
if enable_reward_preview:
    from services.reward_point_system import RewardPointService
    reward_service = RewardPointService()
    reward_data = reward_service.calculate(duration, balance_config)
    result["reward_preview"] = reward_data
```
