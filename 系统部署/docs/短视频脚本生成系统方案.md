# 短视频脚本生成系统方案

## 一、核心理念

### 三大设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                    短视频脚本设计三角原则                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  🎯 前3秒法则                    ⚡ 即时反馈递减               │
│  ┌───┐                            ┌───┐                       │
│  │3秒│ ─── 留住用户 ───→         │奖励│                       │
│  └───┘      必须强钩子             └───┘                       │
│      ↓                                 ↓                        │
│  每3-5秒一个奖励点              前密 → 后疏                    │
│  (前10秒密集)                   (10秒后逐渐拉长)               │
│                                                                  │
│                          ↓                                        │
│  📈 渐进式难度                                                │
│  ┌───┐                                                         │
│  │难度│                                                         │
│  └───┘                                                         │
│      ↓                                                         │
│  低门槛 → 中复杂度 → 高价值结论                                 │
│  (容易理解) (层层递进)   (认知刷新)                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**游戏化内容设计**：借鉴《游戏改变世界》的即时反馈机制，在视频中设置多个"奖励点"，让观众持续获得正向反馈，从而完播、互动、转化。

> 核心逻辑：观众在前3秒获得第一个奖励钩子，之后按照"前密后疏"的节奏持续获得奖励，同时内容难度渐进式提升，最终给出高价值的认知刷新。

### 原则详解

#### 1. 前3秒法则
- **目的**：留住刷到的用户
- **方法**：前3秒必须给出强钩子（痛点/悬念/反差）
- **奖励密度**：每3-5秒一个奖励点

#### 2. 即时反馈递减
- **目的**：适应注意力递减规律
- **方法**：奖励点"前密后疏"分布
- **节奏**：
  | 时间段 | 奖励间隔 | 奖励类型 |
  |--------|---------|---------|
  | 0-10秒 | 3-5秒 | 悬念钩子、痛点冲击 |
  | 10-30秒 | 8-10秒 | 知识干货、情感共鸣 |
  | 30-60秒 | 15秒 | 方案揭晓、价值满足 |
  | 60秒+ | 20-30秒 | 总结升华、认知刷新 |

#### 3. 渐进式难度
- **目的**：让观众"跳一跳够得着"
- **方法**：内容理解难度逐步提升
- **节奏**：
  | 阶段 | 理解难度 | 内容类型 |
  |------|---------|---------|
  | 开场 | 低（容易理解） | 痛点共鸣、简单现象 |
  | 中段 | 中（需要思考） | 原因分析、多角度论证 |
  | 结尾 | 高（认知刷新） | 核心观点、价值升华 |

### 3.2 维度一：业务维度 → 视觉风格

| 业务类型 | 视觉风格 | 色调方案 | 光效建议 |
|---------|---------|---------|---------|
| 实物产品 | 真实场景 + 产品特写 | 暖色调/生活化 | 柔和自然光 |
| 本地服务 | 真实环境 + 人物互动 | 暖白光/家庭感 | 温馨氛围光 |
| 跨区域服务 | 专业场景 + 数据展示 | 冷暖混合/高端感 | 专业影调 |
| 知识付费 | 知识图解 + 人物讲解 | 学术蓝/知识感 | 清晰明亮 |
| 企业服务 | 办公室 + 商务场景 | 深色调/专业感 | 商务影调 |

**奶粉案例**：
- 视觉风格：真实场景（妈妈育儿）+ 产品特写
- 色调：暖色调为主，信任感
- 光效：柔和自然光，母婴用品广告质感

### 3.3 维度二：用户画像维度 → 情绪基调

```
画像判断决策树：

1. 决策周期
   ├── 冲动型（月消费、零食）→ 情绪峰值前置，前3秒必须炸
   ├── 谨慎型（母婴、教育）→ 情绪渐进式，建立信任再转化
   └── 犹豫型（高价、服务）→ 情绪稳定，用专业感替代情绪

2. 信任门槛
   ├── 低门槛（日用品）→ 快速种草，情绪化表达
   ├── 中门槛（食品、健康）→ 先建立信任，再种草
   └── 高门槛（B端、企业）→ 专业权威，理性说服

3. 情感需求
   ├── 理性型（老板、采购）→ 数据说话，逻辑清晰
   ├── 感性型（妈妈、爱宠）→ 情感共鸣，故事驱动
   └── 混合型（家长）→ 情感+理性双线
```

**奶粉案例画像分析**：
- 画像：新手妈妈（决策谨慎型）
- 情绪基调：温暖 + 轻微焦虑
- 信任建立：需要专业背书（成分表、认证）
- 情感触发：宝宝的健康、安全
- 内容节奏：先共鸣（我懂你），再科普（专业），后种草

### 3.4 维度三：内容类型维度 → 节奏结构

| 选题类型 | 节奏结构 | 情绪曲线 | 推荐场景 |
|---------|---------|---------|---------|
| 问题诊断 | 痛点前置 | 高→中→低 | 焦虑共鸣 |
| 解决方案 | 痛点引入→方案→验证 | 低→中→高 | 痛→好奇→爽 |
| 经验分享 | 故事线（起承转合） | 平→高→平 | 平淡→冲突→解决 |
| 避坑指南 | 错误展示→后果→正确 | 高→高→中 | 震惊→紧张→释然 |
| 知识科普 | 提问→揭秘→应用→总结 | 低→中→中→高 | 好奇→揭示→实用 |
| 热点关联 | 热点切入→观点→价值 | 高→平→高 | 抓眼球→沉淀→收获 |

**类型说明**：
- **知识驱动型**：适合高决策成本业务（教育、医疗、房产、金融等），通过知识建立信任
- **价值观驱动型**：适合低决策成本业务（食品、特产、日用品、快消等），通过人设认同建立信任
- **产品驱动型**：适合强品牌背书业务（知名品牌、连锁企业等），信任由机构承担

**信任来源说明**：
- **知识型**：专业内容建立信任，可出镜可不出镜
- **人设型**：人设认同建立信任，**必须出镜**
- **机构型**：品牌背书建立信任，**不需要出镜**

**注**：观点输出类可衍生深度递进型（参考@认知便利M），通过均衡器调节问题悬念(70-90%)和信息密度(70-90%)实现。

#### 人设价值观类（低决策成本业务专用）

对于**决策成本低、替代品多、影响周期短、挽回成本低**的业务（如矿泉水、特产零食、日用品），不适合走知识干货路线，而应该走**人设价值观驱动**的路线。

```
┌─────────────────────────────────────────────────────────────────┐
│                  价值观驱动型内容 vs 知识驱动型内容                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  传统知识型：知识 → 信任 → 转化   (适合高决策成本业务)            │
│                                                                  │
│  人设价值观型：认同人 → 信任人 → 转化  (适合低决策成本业务)        │
│                  ↑                                               │
│              观点/态度/冲突/吐槽                                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**适用业务特征**：

| 业务特征 | 说明 | 适合人设内容 |
|---------|------|-------------|
| 决策成本低 | 不需要太多思考比较 | 跟着人走 |
| 客户选择多 | 替代品多、差异化难 | 人设即差异 |
| 影响周期短 | 复购快、决策快 | 快速种草 |
| 挽回成本低 | 流失快但获取也快 | 情感粘性 |

**人设价值观类的核心要素**：

```
1. 话题钩子（替代痛点钩子）
   ├── 抛出观点："我觉得XX是最大的智商税"
   ├── 制造冲突："为什么你买的XX总是踩坑"
   ├── 制造反差："卖山货5年，第一次遇到这种事"
   └── 吐槽引入："今天必须吐槽一下这个行业"

2. 奖励点类型（替代知识奖励）
   ├── 观点奖励 → 引发认同感
   ├── 冲突展开 → 引发好奇心
   ├── 金句提炼 → 引发分享欲
   └── 价值观升华 → 强化记忆

3. 难度递进（替代知识难度）
   ├── 低门槛：简单现象/吐槽开场
   ├── 中门槛：原因分析/多角度解读
   └── 高门槛：价值观输出/认知刷新
```

**人设价值观类细分类型**：

| 类型 | 钩子 | 奖励点 | 难度曲线 | 典型话题 |
|------|------|--------|---------|---------|
| **观点输出型** | 争议观点 | 金句、认同感 | 平→高 | "我认为XX..." |
| **冲突叙事型** | 事件冲突 | 故事性、悬念 | 平→高→更高 | 客户冲突、行业内幕 |
| **吐槽犀利型** | 行业痛点 | 共鸣、反差 | 高→更高 | 行业吐槽、避坑 |
| **推崇种草型** | 产品故事 | 情感满足、价值感 | 平→中→高 | 推崇某个产品/理念 |
| **价值观型** | 理念冲突 | 认知刷新 | 低→高→极高 | 人生观、职业观 |

**人设价值观类专属均衡器配置**：

```
人设价值观类（新增预设）

├── 信息密度：15-25%  (几乎不输出知识)
├── 问题悬念：60-80%  (话题悬念)
├── 情绪波动：70-85%  (高情绪输出)
├── 互动频率：60-75%  (引发讨论)
├── 奖励分布：50-60%  (均匀偏前)
└── 难度递进：60-75%  (认知深化)
```

**注**：与知识干货类的区别在于——知识干货类用"信息密度"作为主要参数，人设价值观类用"情绪波动"和"问题悬念"作为主要参数。

### 3.5 维度四：人设维度 → 语气钩子

| 人设 | 语气风格 | 钩子类型 | 适用画像 |
|-----|---------|---------|---------|
| 陪伴者 | 温暖、共情 | 痛点+情感 | 焦虑型、迷茫型 |
| 教导者 | 专业、权威 | 知识+数字 | 理性型、谨慎型 |
| 崇拜者 | 热情、种草 | 效果+对比 | 追求型、跟风型 |
| 陪衬者 | 真实、共鸣 | 故事+反转 | 经验型、分享型 |

## 四、内容价值三角：有趣、有用、有共鸣

商业核心是解决信任问题。观众愿意看内容，是因为**有用、有趣、有共鸣**。这三个要素是内容的价值基础。

### 4.1 三要素定位与作用

| 要素 | 核心问题 | 目的 | 时间位置 |
|------|---------|------|---------|
| **有趣** | "为什么要看？" | 留住人，前3秒抓住注意力 | 视频前段 |
| **有用** | "学到了什么？" | 提供价值，让观众觉得没白看 | 视频中段 |
| **有共鸣** | "为什么关注他？" | 带走情感，认可这个人 | 视频后段 |

### 4.2 三要素具体策略

#### 有趣（留住人）

**目的**：前3秒抓住注意力，让观众不想划走

| 策略 | 具体做法 | 案例 |
|------|---------|------|
| **反常识** | 打破常规认知 | "卖了几千单，第一次被顾客骂哭了" |
| **冲突感** | 制造对立或反差 | "同行卖29，我坚持卖39" |
| **悬念感** | 留个钩子 | "直到发生这件事，我才知道..." |
| **情绪感** | 强烈的情绪表达 | "我真的忍不了，必须说几句" |
| **真实感** | 不完美的真实 | "今天翻车了，给大家看看" |

#### 有用（提供价值）

**目的**：让观众觉得"学到了"、"记住了"

| 策略 | 适用场景 | 说明 |
|------|---------|------|
| **知识型有用** | 教育、医疗、金融 | 技巧、方法、揭秘 |
| **经验型有用** | 食品、特产、日用品 | 避坑、对比、选购指南 |
| **情感型有用** | 人设型账号 | 观点、态度、价值观 |
| **产品型有用** | 机构型账号 | 品质、规格、售后保障 |

**关键点**：每30秒给一个"干货点"，用数字、对比、举例让价值可感知，结论先行。

#### 有共鸣（带走情感）

**目的**：让观众"认可你这个人"，不只是"这个内容"

| 策略 | 具体做法 | 触发情感 |
|------|---------|---------|
| **说出观众想说** | "你是不是也遇到过..." | 共情 |
| **替观众表态** | "我就想不通，为什么..." | 认同 |
| **展示真实一面** | 失误、困难、坚持 | 信任 |
| **价值观输出** | "我认为..."、"我的原则是..." | 认同 |
| **金句结尾** | 一句话总结核心观点 | 记忆 |

**关键点**：结尾要有"一句话"，让观众能转述你的观点，留下"这就是XX"的记忆点。

### 4.3 三要素在视频中的时间分布

```
0-3秒    3-10秒    10-30秒   30-60秒   60秒+
 ┌───┐    ┌───┐     ┌───┐     ┌───┐    ┌───┐
 │有趣│ → │有趣│  → │有用│  → │共鸣│ → │金句│
 └───┘    └───┘     └───┘     └───┘    └───┘
  ↑                        ↑            ↑
 钩子                      干货          认同
 悬念                      知识          观点
 冲突                      经验          价值观
```

### 4.4 选题类型与三要素配比

| 选题类型 | 有趣(留人) | 有用(价值) | 有共鸣(认同) |
|---------|-----------|-----------|-------------|
| 问题诊断类 | 30% | **50%** | 20% |
| 解决方案类 | 20% | **60%** | 20% |
| 案例分享类 | 30% | 40% | **30%** |
| 产品推荐类 | 40% | 40% | 20% |
| 知识科普类 | 20% | **60%** | 20% |
| 热点关联类 | **50%** | 30% | 20% |
| 人设故事类 | 30% | 20% | **50%** |
| **人设价值观类** | 30% | 15% | **55%** |
| 观点输出类 | 20% | 30% | **50%** |
| 机构产品类 | 20% | **60%** | 20% |

### 4.5 三要素与均衡器参数映射

| 三要素 | 对应均衡器参数 | 说明 |
|-------|---------------|------|
| **有趣** | 问题悬念 + 情绪波动 | 悬念感=有趣，情绪起伏=有趣 |
| **有用** | 信息密度 + 难度递进 | 信息量=有用，层层递进=干货感 |
| **有共鸣** | 情绪波动 + 互动频率 | 情绪共鸣=认同感，互动=参与感 |

### 4.2 两种出镜模式对比

| 维度 | 完全不出镜 | 出镜（数字人） |
|-----|----------|--------------|
| **内容风格** | 旁白为主，画面灵活 | 主播讲解，沉浸感强 |
| **脚本写法** | "旁白：..." "画面：..." | "主播：..." "画面：..." |
| **互动方式** | 声音引导 + 画面互动 | 主播面对面互动，更强 |
| **信任建立** | 需要其他元素（口碑/数据） | 主播形象直接建立信任 |
| **制作成本** | 低（只需配音） | 中高（数字人定制） |
| **适用场景** | 知识科普、经验分享 | 带货、种草、答疑 |

## 七、评分体系

### 7.1 总分计算公式

```
总分 = 情绪得分 × 30% + 节奏得分 × 30% + 互动得分 × 40%
```

### 7.2 情绪评分（0-100分）

| 评分维度 | 分值 | 优秀标准 | 合格标准 |
|---------|-----|---------|---------|
| 情绪词密度 | 25分 | 每15秒1个情绪词 | 每30秒1个情绪词 |
| 情绪词多样性 | 25分 | 使用3种以上情绪词 | 使用2种情绪词 |
| 情绪转折设计 | 25分 | 有明显情绪起伏 | 有1-2个情绪变化 |
| 代入感设计 | 25分 | 多次使用"你"、设问 | 偶尔使用"你" |

### 7.3 节奏评分（0-100分）

| 评分维度 | 分值 | 优秀标准 | 合格标准 |
|---------|-----|---------|---------|
| 钩子数量 | 30分 | 达到时长推荐数量 | 达到推荐的80% |
| 钩子时机 | 30分 | 前3秒有钩子 + 中间有钩子 | 前3秒有钩子 |
| 节奏峰值 | 20分 | 每个节奏段落有1个峰值点 | 有2-3个峰值点 |
| 时长匹配度 | 20分 | 时长与内容饱满度匹配 | 时长略长或略短 |

### 7.4 互动评分（0-100分）

| 评分维度 | 分值 | 优秀标准 | 合格标准 |
|---------|-----|---------|---------|
| 互动点数量 | 30分 | 达到推荐的互动点数量 | 达到推荐的60% |
| 互动点时机 | 30分 | 互动点设计在价值点后 | 互动点时机合理 |
| CTA多样性 | 20分 | 使用3种以上CTA类型 | 使用2种CTA类型 |
| 引导自然度 | 20分 | 互动引导融入内容，不生硬 | 引导略生硬 |

### 12.3 后端服务层逻辑

```python
# services/video_script_service.py

class VideoScriptService:

    def generate_script(self, request: ScriptGenerateRequest) -> ScriptResponse:
        """
        脚本生成主流程
        """
        # 1. 获取客户画像
        customer = self.get_customer_profile(request.customer_id)

        # 2. 获取选题类型配置
        topic_config = self.get_topic_config(request.video_config.topic_type)

        # 3. 判断内容逻辑类型
        content_logic = topic_config.get('content_logic', 'knowledge')

        # 4. 合并均衡器配置（根据内容逻辑类型调整默认参数）
        balancer = self.merge_balancer_config(
            request.balancer_config,
            topic_config,
            content_logic  # 新增：传递内容逻辑类型
        )

        # 5. 计算奖励点分布
        reward_plan = self.calculate_reward_distribution(
            total_duration=request.video_config.duration,
            distribution_rate=balancer.reward_distribution,
            content_logic=content_logic  # 新增：价值观类奖励点不同
        )

        # 6. 计算钩子策略
        hook_strategy = self.get_hook_strategy(
            topic_config,  # 获取钩子类型
            content_logic
        )

        # 7. 计算难度曲线
        difficulty_curve = self.calculate_difficulty_curve(
            total_duration=request.video_config.duration,
            progression_rate=balancer.difficulty_progression,
            content_logic=content_logic  # 新增：价值观类难度曲线不同
        )

        # 8. 生成脚本结构
        script_structure = self.build_script_structure(
            topic=request.video_config.topic_title,
            topic_type=request.video_config.topic_type,
            content_logic=content_logic,  # 新增
            hook_strategy=hook_strategy,  # 新增
            balancer=balancer,
            reward_plan=reward_plan,
            difficulty_curve=difficulty_curve,
            customer=customer
        )

        # 9. 填充内容（根据内容逻辑选择不同的内容生成策略）
        full_script = self.fill_script_content(
            structure=script_structure,
            customer=customer,
            keywords=request.keyword_context,
            content_logic=content_logic  # 新增
        )

        # 10. 评分
        score_report = self.score_script(full_script, content_logic)

        # 11. 保存记录
        record = self.save_record(request, full_script, score_report)

        return ScriptResponse(record_id=record.id, script=full_script, score=score_report)

    def get_hook_strategy(self, topic_config, content_logic):
        """
        根据内容逻辑类型获取钩子策略
        """
        hook_type = topic_config.get('hook_type', 'pain_point')

        if content_logic == 'values':
            # 价值观驱动型：话题钩子
            return {
                "hook_type": "topic_point",
                "hook_templates": [
                    "我觉得{}",  # 观点抛出
                    "为什么你{}",  # 冲突制造
                    "卖{}年，第一次遇到{}",  # 反差制造
                    "今天必须吐槽一下{}",  # 吐槽引入
                ],
                "avoid_pain_point": True,  # 不使用痛点钩子
            }
        else:
            # 知识驱动型：痛点钩子
            return {
                "hook_type": "pain_point",
                "hook_templates": [
                    "你是不是也在为{}烦恼？",
                    "{}的人注意了！",
                    "{}的真相是...",
                ],
                "avoid_pain_point": False,
            }

    def get_reward_type(self, content_logic):
        """
        根据内容逻辑类型返回不同的奖励点类型
        """
        if content_logic == 'values':
            # 价值观驱动型：观点奖励、情绪奖励
            return {
                "primary": ["观点输出", "金句提炼", "价值观升华"],
                "secondary": ["情感共鸣", "冲突揭晓", "认知刷新"],
                "avoid_knowledge": True,  # 避免知识奖励
            }
        else:
            # 知识驱动型：知识奖励
            return {
                "primary": ["知识揭秘", "技巧分享", "干货输出"],
                "secondary": ["案例展示", "效果验证", "方案提供"],
                "avoid_knowledge": False,
            }

    def calculate_reward_distribution(self, total_duration: int, distribution_rate: int):
        """
        计算奖励点分布 - 前密后疏算法
        distribution_rate: 30-100 (越大越前密)
        """
        # 基础奖励点数量（按时长估算）
        base_count = max(3, total_duration // 10)

        # 时间段划分
        segments = [
            {"range": (0, 10), "weight": 0},   # 前10秒
            {"range": (10, 30), "weight": 0},   # 10-30秒
            {"range": (30, 60), "weight": 0},   # 30-60秒
            {"range": (60, 999), "weight": 0},  # 60秒以上
        ]

        if distribution_rate >= 80:  # 强烈前密
            weights = [0.5, 0.3, 0.15, 0.05]
        elif distribution_rate >= 50:  # 标准前密
            weights = [0.35, 0.35, 0.2, 0.1]
        else:  # 均匀分布
            weights = [0.2, 0.3, 0.3, 0.2]

        for seg, w in zip(segments, weights):
            seg["count"] = max(1, int(base_count * w))

        return {
            "segments": segments,
            "interval_pattern": self.generate_interval_pattern(segments, total_duration)
        }

    def calculate_difficulty_curve(self, total_duration: int, progression_rate: int):
        """
        计算难度递进曲线
        progression_rate: 30-100 (越大递进越快)
        """
        checkpoints = [
            {"position": 0.0, "difficulty": 20},   # 开场低难度
            {"position": 0.3, "difficulty": 40},   # 中段提升
            {"position": 0.7, "difficulty": 70},   # 后段中高
            {"position": 1.0, "difficulty": 100},  # 结尾高难度
        ]

        # 根据递进速率调整中间点
        if progression_rate >= 80:  # 快速递进
            checkpoints[1]["difficulty"] = 60
            checkpoints[2]["difficulty"] = 90
        elif progression_rate >= 50:  # 标准递进
            checkpoints[1]["difficulty"] = 50
            checkpoints[2]["difficulty"] = 75
        else:  # 平缓递进
            checkpoints[1]["difficulty"] = 40
            checkpoints[2]["difficulty"] = 60

        return checkpoints

    def score_script(self, script: Script) -> ScoreReport:
        """
        脚本评分逻辑
        """
        scores = []

        # 1. 前3秒钩子评分
        hook_score = self.score_hook(script.segments[0] if script.segments else None)
        scores.append({"name": "前3秒钩子", "score": hook_score, "weight": 0.25})

        # 2. 奖励点分布评分
        reward_score = self.score_reward_distribution(
            script.reward_distribution,
            script.total_duration,
            script.balancer_config.reward_distribution
        )
        scores.append({"name": "奖励点分布", "score": reward_score, "weight": 0.2})

        # 3. 难度递进评分
        difficulty_score = self.score_difficulty_progression(
            script.difficulty_curve,
            script.balancer_config.difficulty_progression
        )
        scores.append({"name": "难度递进", "score": difficulty_score, "weight": 0.2})

        # 4. 情绪曲线评分
        emotion_score = self.score_emotion_curve(script)
        scores.append({"name": "情绪曲线", "score": emotion_score, "weight": 0.2})

        # 5. 互动引导评分
        interaction_score = self.score_interaction(script)
        scores.append({"name": "互动引导", "score": interaction_score, "weight": 0.15})

        # 计算总分
        total = sum(s["score"] * s["weight"] for s in scores)

        return ScoreReport(total_score=total, dimensions=scores)
```

### 14.8 技术实现

#### 14.8.1 服务层设计

```python
# services/search_service.py

from typing import List, Optional
from dataclasses import dataclass
from enum import Enum
import redis
import hashlib
from datetime import datetime, timedelta

class ReferenceType(Enum):
    QUOTE = "quote"        # 名言引用
    DATA = "data"          # 数据佐证
    CASE = "case"          # 案例引用
    HOT = "hot"            # 热点关联
    TERM = "term"          # 术语解释

@dataclass
class SearchReference:
    """搜索引用结果"""
    ref_type: ReferenceType
    content: str
    source_name: str
    source_url: str
    author: Optional[str] = None
    publish_date: Optional[str] = None
    search_query: str
    confidence: float = 0.0

class SearchService:
    """实时搜索服务"""
    
    def __init__(self, tavily_key: str, redis_client: redis.Redis):
        self.client = TavilyClient(api_key=tavily_key)
        self.redis = redis_client
    
    async def search_reference(
        self,
        queries: List[str],
        ref_type: ReferenceType,
        cache_ttl: int = 86400 * 7
    ) -> List[SearchReference]:
        """
        搜索引用内容
        
        Args:
            queries: 搜索词列表
            ref_type: 引用类型
            cache_ttl: 缓存有效期（秒）
        """
        results = []
        
        for query in queries:
            # 1. 检查缓存
            cache_key = self._get_cache_key(query, ref_type)
            cached = await self._get_from_cache(cache_key)
            if cached:
                results.extend(cached)
                continue
            
            # 2. 调用搜索API
            search_result = await self._call_search_api(query, ref_type)
            
            # 3. 格式化结果
            references = self._format_references(search_result, query, ref_type)
            
            # 4. 写入缓存
            await self._save_to_cache(cache_key, references, cache_ttl)
            
            results.extend(references)
        
        return results
    
    async def _call_search_api(self, query: str, ref_type: ReferenceType):
        """调用搜索API"""
        search_depth = "advanced"
        
        # 根据引用类型调整搜索策略
        if ref_type == ReferenceType.QUOTE:
            search_depth = "basic"  # 名言类简单搜索即可
        elif ref_type == ReferenceType.DATA:
            search_depth = "advanced"  # 数据类需要深度搜索
        
        return self.client.search(
            query=query,
            search_depth=search_depth,
            include_answer=True,
            max_results=3
        )
    
    def _format_references(
        self, 
        result: dict, 
        query: str, 
        ref_type: ReferenceType
    ) -> List[SearchReference]:
        """格式化搜索结果为引用"""
        references = []
        
        for item in result.get("results", []):
            ref = SearchReference(
                ref_type=ref_type,
                content=item.get("content", ""),
                source_name=item.get("title", ""),
                source_url=item.get("url", ""),
                search_query=query,
                confidence=item.get("score", 0.0)
            )
            references.append(ref)
        
        return references
    
    def _get_cache_key(self, query: str, ref_type: ReferenceType) -> str:
        """生成缓存键"""
        query_hash = hashlib.md5(query.encode()).hexdigest()
        return f"search:{ref_type.value}:{query_hash}"
```

#### 14.8.2 生成引擎集成

```python
# services/script_generator.py

class ScriptGenerator:
    """脚本生成引擎 - 集成搜索能力"""
    
    def __init__(
        self,
        llm_service: LLMService,
        search_service: SearchService
    ):
        self.llm = llm_service
        self.search = search_service
    
    async def generate_script(self, request: GenerateRequest) -> ScriptResult:
        """
        生成脚本 - 包含搜索增强
        """
        # 1. 分析是否需要搜索引用
        search_needs = self._analyze_search_needs(request)
        
        # 2. 如果需要，触发搜索
        references = []
        if search_needs:
            references = await self._fetch_references(search_needs)
        
        # 3. 构建增强提示词
        enhanced_prompt = self._build_enhanced_prompt(
            request, 
            references
        )
        
        # 4. 调用LLM生成
        script = await self.llm.generate(enhanced_prompt)
        
        # 5. 后处理：插入引用标注
        script = self._insert_reference_annotations(script, references)
        
        # 6. 保存引用记录
        await self._save_reference_records(script.id, references)
        
        return script
    
    def _analyze_search_needs(self, request: GenerateRequest) -> List[dict]:
        """
        分析搜索需求
        
        返回:
            List[{
                "type": ReferenceType,
                "keywords": List[str],
                "priority": int
            }]
        """
        needs = []
        
        # 根据选题类型判断
        if request.topic_type in ["知识科普", "解决方案"]:
            needs.append({
                "type": ReferenceType.DATA,
                "keywords": self._extract_data_keywords(request.topic),
                "priority": 1
            })
        
        # 根据内容关键词判断
        keyword_mapping = {
            "名言": ReferenceType.QUOTE,
            "数据": ReferenceType.DATA,
            "案例": ReferenceType.CASE,
            "热点": ReferenceType.HOT,
            "术语": ReferenceType.TERM,
        }
        
        for keyword, ref_type in keyword_mapping.items():
            if keyword in request.topic:
                needs.append({
                    "type": ref_type,
                    "keywords": self._extract_keywords(request.topic),
                    "priority": 2
                })
        
        return needs
    
    async def _fetch_references(self, needs: List[dict]) -> List[SearchReference]:
        """获取引用内容"""
        all_references = []
        
        for need in needs:
            refs = await self.search.search_reference(
                queries=need["keywords"],
                ref_type=need["type"]
            )
            all_references.extend(refs)
        
        # 去重 + 排序
        return self._deduplicate_and_sort(all_references)
    
    def _build_enhanced_prompt(
        self, 
        request: GenerateRequest,
        references: List[SearchReference]
    ) -> str:
        """构建增强提示词"""
        
        # 引用上下文
        ref_context = ""
        if references:
            ref_context = "\n\n【参考资料】\n"
            for i, ref in enumerate(references[:5], 1):  # 最多5个引用
                ref_context += f"{i}. {ref.content}\n"
                ref_context += f"   来源：{ref.source_name}\n\n"
        
        prompt = f"""
请基于以下需求生成短视频脚本：

【选题】{request.topic}
【时长】{request.duration}秒
【IP人设】{request.ip_config}
【风格】{request.style}

{ref_context}

要求：
1. 适当融入上述参考资料的内容
2. 引用时标注来源（来源名称 + 网址）
3. 保持脚本的流畅性和说服力
"""
        return prompt
    
    def _insert_reference_annotations(
        self, 
        script: Script,
        references: List[SearchReference]
    ) -> Script:
        """在脚本中插入引用标注"""
        # 关联引用与脚本段落
        for segment in script.segments:
            if segment.has_reference:
                ref = self._find_best_reference(segment, references)
                if ref:
                    segment.reference = ref
        
        return script
```

### 14.11 错误处理与降级策略

#### 14.11.1 降级原则

**核心原则**：搜索失败不影响脚本生成

| 故障场景 | 处理策略 | 用户感知 |
|---------|---------|---------|
| **搜索超时** | 直接降级，继续生成 | 无感知 |
| **API返回错误** | 降级 + 记录日志 | 无感知 |
| **结果为空** | 跳过该引用，继续生成 | 脚本无引用 |
| **部分失败** | 成功的部分使用，失败的跳过 | 引用减少 |
| **服务不可用** | 降级 + 告警通知 | 无感知 |

```python
        result = await tavily_client.search(queries)

        # 缓存结果
        await save_to_cache(queries, result)

        logger.info(
            "search_completed",
            queries=queries,
            result_count=len(result.get("results", [])),
            duration_ms=(time.time() - start_time) * 1000
        )

        return result

    except Exception as e:
        logger.error(
            "search_failed",
            queries=queries,
            error=str(e),
            error_type=type(e).__name__
        )
        raise
```
