# 前端开发规范（UI 可读性相关）

## 文字可读性

1. **禁止使用深色底色的徽章类样式显示正文信息或统计数字**
   - 包括但不限于：`badge bg-primary`、`badge bg-secondary`、`badge bg-success` 等。
   - 统计数字、标签、计数器，统一使用：`text-muted small` 或 `text-muted small border rounded px-2 py-1` 等**无实色背景**样式。

2. **可点击按钮才允许使用实色背景**
   - 按钮使用 `btn-*` 系列，例如：`btn-primary`、`btn-success`。
   - 纯展示型文字统一避免有色背景，仅用文字颜色和细边框区分层级。

3. **Badge 组件必须使用 `text-bg-*` 而非 `bg-*`**
   - Bootstrap 5 的 badge 有两种写法：
     - `badge bg-success` - 只设置背景色，文字颜色继承父元素
     - `text-bg-success` - 同时设置背景色和文字颜色，确保可读性
   - **暗黑模式下**：`bg-*` 可能导致文字看不清楚，必须使用 `text-bg-*`
   - 示例：
     ```html
     <!-- 错误：暗黑模式下文字可能看不清 -->
     <span class="badge bg-success">8分</span>

     <!-- 正确：自动适配暗黑模式 -->
     <span class="badge text-bg-success">8分</span>
     ```

4. **以后新页面/组件必须遵守以上规则**
   - 发现历史代码中有使用 `badge bg-*` 显示正文或数字的场景，优先改为：
     - 统计：`<span class="text-muted small">xx 条规则</span>`
     - 标签：`<span class="text-muted small border rounded px-2 py-1">标签</span>`
     - 评分：`<span class="badge text-bg-success">8分</span>`

## 暗黑模式兼容性

1. **避免使用硬编码的颜色**
   - 使用 CSS 变量：`var(--color-bg-primary)`、`var(--text-muted)` 等
   - 避免使用纯黑色 `#000` 或纯白色 `#fff` 作为背景

2. **图片和图标注意对比度**
   - 确保图标在不同主题下都清晰可见
   - 使用 `text-*` 类而非硬编码颜色
