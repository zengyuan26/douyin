# 内容框架方法论

**版本：2.0**
**生成日期：2026年4月24日**
**核心思维：不卖货，先锁人 → 放大痛点 → 方案对比 → 愿景勾画 → 消除顾虑**
**新增模块：供需失衡框架——好选题从"问题严重且没人解决"中来**

---

## 核心理念

> 选题不是一上来就卖货

传统内容思路：讲功能 → 说好处 → 促转化

新框架内容思路：
1. 先把受众卡死，告诉对方"这是专门给某种人群准备的"
2. 目的不是转化，而是让看到内容的人**立刻判断这东西和自己有没有关**
3. 再进入痛点 → 方案 → 愿景 → 行动的内容循环

---

## 新增：供需失衡——好选题的底层逻辑

> 💡 **好选题的本质**：用户有真实问题 + 现有供给没很好解决 + 你的内容能填补这个空白

### 供需失衡 × 五段式的对应关系

```
供需失衡维度                    五段式内容框架
──────────────────────────────────────────────────────
severity（严重程度）    ←→   第二阶段：痛点放大（讲清不解决的代价）
urgency（紧急程度）     ←→   第五阶段：顾虑消除（现在做/不做的时间压力）
why_unsolved（未解决原因）←→  第二阶段：痛点放大（讲清为什么现有方案差）
supply_gap（供给缺口）  ←→   第三阶段：方案对比（讲清我的方案好在哪）
entry_angle（切入角度） ←→   第四阶段：愿景勾画（用完之后的变化）
```

**severity × urgency → 内容优先级**

| severity × urgency | 内容策略 |
|---------------------|----------|
| 高 + 高 | 立即做！流量+转化双收 |
| 高 + 中/低 | 旺季前做，长期蓄水 |
| 中 + 高 | 热点时快速收割 |
| 中 + 中 | 持续稳定输出 |
| 中 + 低 | 按需制作 |

## 五段式内容框架

### 第一阶段：受众锁定

**目的**：让人立刻判断"这说的是不是我"

**核心思维**：
- 不讲功能，只讲"你是谁"
- 用人群特征卡死受众
- 生成精准画像，让目标人群产生"这说的就是我"的共鸣

**关键词类型**：
- 受众锁定关键词
- 人群画像词
- 人群特征词

**内容方向**：
- "你是这种人吗？"
- "这几种人注意了"
- "{人群标签}+{核心业务}"组合

**示例**：
> 如果是灌香肠业务：
> "农村过年灌香肠，这几种人最容易踩坑——看看你有没有中招"
> "家里老人每年都要灌香肠的年轻人，这篇文章专门写给你们"

---

### 第二阶段：痛点放大

**目的**：让人意识到"现在的做法有多糟糕"

**核心思维**：
- 讲问题不急着讲功能
- 这一封更像是在放大用户痛点
- 重点不是"有哪些功能"，而是"你现在的方式有多麻烦"
- 让人意识到自己现在的解决方案有多低效或者错误
- 这个产品为什么有存在价值

**关键词类型**：
- 问题现状词（常见问题、难点痛点）
- 错误认知词（你以为对的其实是错的）
- 低效方案词（自己做的代价有多大）

**内容方向**：
- "你正在经历这些问题"
- "你以为对的，其实是错的"
- "自己做的代价有多大"

**示例**：
> "灌香肠看似简单，90%的人都做错了这三步"
> "自己灌香肠费力不讨好，白忙活一场"
> "你以为灌香肠很难，其实是你方法用错了"

---

### 第三阶段：方案对比

**目的**：突出"我的方案为什么更好"

**核心思维**：
- 分析问题，对比其他解决方案
- 突出方案的差异化
- 解决步骤、价值、好处

**关键词类型**：
- 差异化对比词（A vs B、哪个更好）
- 方案优势词（省心、省钱、省时）
- 信任佐证词（经验、案例、过程透明）

**内容方向**：
- "我们和别人的区别在哪里"
- "选我们能解决这些问题"
- "你放心的理由是什么"

**示例**：
> "找了4家灌香肠，终于知道为什么有人做出来不好吃"
> "专业灌香肠和自己在家的区别，看完你就懂了"
> "22年老手艺人和新手灌的香肠有什么区别"

---

### 第四阶段：愿景勾画

**目的**：让用户代入"用之后会变多好"

**核心思维**：
- 讲愿景，讲使用后的好处
- 改变，讲使用场景
- 让用户脑子里开始代入自己用了之后会变得多好

**关键词类型**：
- 使用好处词（有多爽、有多舒服）
- 改变场景词（前和后对比）

**内容方向**：
- "用完之后你会发现..."
- "以前是这样，现在是这样"

**示例**：
> "用了这个方法灌香肠，再也不用羡慕别人家的味道"
> "灌香肠前和灌香肠后，家里的变化让人惊喜"
> "自从找了专业师傅灌香肠，过年再也没操过心"

---

### 第五阶段：顾虑消除

**目的**：打消"用了之后有问题怎么办"

**核心思维**：
- 对使用中可能出现的延伸问题进行展开
- 打消行动顾虑，补充方案
- 进入下一个内容制作循环

**关键词类型**：
- 延伸问题词（后期问题、售后问题）
- 行动打消词（现在做晚不晚、值不值得）

**内容方向**：
- "后期遇到问题怎么办"
- "现在做来得及吗"

**示例**：
> "灌香肠做好了能放多久？保存方法全在这"
> "现在灌香肠还来得及吗？年前最后几天攻略"
> "灌香肠不满意可以重做吗？说说我的售后承诺"

## 账号阶段递进规则

### 判定原则：双轨制

**时间维度（参考基准）+ 量化指标（核心触发）= 先到者决定**

时间仅作为参考基准，量化指标达到任一触发条件即进入下一阶段。

---

### 阶段递进触发条件

#### 起号期 → 成长期

| 触发维度 | 具体指标 | 触发方式 |
|---------|---------|---------|
| 内容生成量 | 累计发布图文/视频 ≥ **30篇** | 满足任一条件即进入成长期 |
| 时间基准 | 自首次发布起 ≥ **3个月** | |
| 粉丝规模 | 粉丝数 ≥ **500** | |
| 核心指标 | 有互动（点赞+评论+收藏≥1）的笔记占比 ≥ **30%** | |

#### 成长期 → 成熟期

| 触发维度 | 具体指标 | 触发方式 |
|---------|---------|---------|
| 内容生成量 | 累计发布图文/视频 ≥ **120篇** | 满足任一条件即进入成熟期 |
| 时间基准 | 自首次发布起 ≥ **6个月** | |
| 粉丝规模 | 粉丝数 ≥ **3000** | |
| 核心指标 | 单篇平均互动量 ≥ **50**（点赞+评论+收藏） | |

---

### 阶段停留规则

- **降级警戒**：若连续 **30天** 无新内容产出，阶段自动降一级（成熟期→成长期→起号期）
- **季节性业务豁免**：季节性业务（如灌香肠）在淡季期间暂停计时，淡季结束后重新计时
- **强制升级审查**：无论是否达到量化指标，内容生产超过 **9个月** 仍未进入成熟期，由运营专家介入诊断

---

### 各阶段核心任务对照

| 阶段 | 核心任务 | 阶段特征 |
|------|---------|---------|
| 起号期 | 建认知、养标签、积累种子用户 | 内容结构关键词占比高，以受众锁定+痛点放大为主 |
| 成长期 | 扩影响、建信任、测试变现路径 | 两类关键词均衡，逐步引入方案对比+愿景勾画 |
| 成熟期 | 促转化、提效率、打造内容闭环 | 传统搜索关键词占比高，以顾虑消除+直接需求内容为主 |
