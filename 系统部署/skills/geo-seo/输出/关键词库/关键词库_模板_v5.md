## 📋 关键词库说明

### 分类逻辑

> 💡 本关键词库采用**人群分类**模式，按用户画像（人群）分组：
> - 每个画像对应一个**问题类型**
> - 每个画像下的关键词服务于该画像的**具体场景**

```
┌─────────────────────────────────────────────────────────────┐
│  关键词库（总库）                                          │
│  ├─ 📦 公用关键词库（所有画像共享）                        │
│  │   └─ 跨画像的通用关键词                                │
│  │                                                         │
│  └─ 📦 个性化关键词库（每个画像独有）                      │
│      ├─ 画像A（问题类型1）→ 画像A专属关键词                │
│      ├─ 画像B（问题类型2）→ 画像B专属关键词                │
│      └─ 画像C（问题类型3）→ 画像C专属关键词                │
└─────────────────────────────────────────────────────────────┘
```

### 付费者与使用者判断

> ⚠️ **重要**：先判断"付费者与使用者是否分离"

| 情况 | 示例 | 关键词库结构 |
|------|------|--------------|
| **付费者=使用者** | 灌香肠加工、建材销售、本地服务 | 按本模板，公用+个性化 |
| **付费者≠使用者** | 奶粉、童装、礼品、养老服务 | 分离分类：使用者问题+付费者顾虑 |

### 更新场景

| 场景 | 触发条件 | 更新方式 | 影响范围 |
|------|----------|----------|----------|
| 蓝海机会变了 | 业务方向调整 | 全量替换 | 所有画像 |
| 新增画像 | 新问题类型 | 增量添加 | 仅新画像 |
| 删除画像 | 画像失效 | 清理关键词 | 仅该画像 |
| 画像效果差 | 关键词效果不佳 | 局部替换 | 仅该画像 |

### 更新操作

| 操作 | 影响范围 | 说明 |
|------|----------|------|
| 更新公用关键词 | 所有画像 | 跨画像的通用关键词 |
| 更新画像A专属关键词 | 仅画像A | 该画像的关键词 |
| 新增画像关键词 | 新画像 | 新画像的关键词 |
| 删除画像关键词 | 被删除画像 | 清理该画像关键词 |
