> ⚠️ **痛点原则（核心！所有关键词必须遵循）**：
> - ✅ 是不是客户**真实想要**的？（不是我们以为的）
> - ✅ 是不是客户的**痛点**？（不解决会难受）
> - ✅ 是不是**紧急**的？（不解决会出问题）
> - ❌ 不要自嗨！不要写客户根本不关心的内容！

> ⚠️ **数量要求**：关键词库必须生成 **至少150个** 关键词，覆盖所有分类

## 📋 关键词库说明

### 分类逻辑

> 💡 本关键词库采用**双维度分类**，确保关键词既服务于内容框架，又覆盖业务全链路

```
┌─────────────────────────────────────────────────────────────┐
│  维度一：内容结构维度（用户认知阶段）                        │
│  ├─ 受众锁定 → 让人判断"这说的是不是我"                     │
│  ├─ 痛点放大 → 让人意识到"现在有多糟糕"                     │
│  ├─ 方案对比 → 让人知道"为什么选我"                         │
│  ├─ 愿景勾画 → 让人期待"用后会变多好"                       │
│  └─ 顾虑消除 → 打消"用了之后怎么办"                         │
│                                                             │
│  维度二：业务覆盖维度（关键词覆盖广度）                       │
│  ├─ 直接需求 → 用户直接找服务/产品                           │
│  ├─ 场景需求 → 特定场景下的需求                             │
│  ├─ 行业上下游 → 上游材料/下游服务                          │
│  ├─ 配套方案 → 搭配使用的关联产品/服务                       │
│  ├─ 地域需求 → 本地化搜索                                   │
│  ├─ 技巧干货 → 知识型搜索                                   │
│  └─ 颠覆认知 → 反常识/辟谣类                                │
└─────────────────────────────────────────────────────────────┘
```

## 📋 付费者与使用者判断

> ⚠️ **重要**：先判断"付费者与使用者是否分离"，再选择分类逻辑

| 情况 | 示例 | 关键词库结构 |
|------|------|--------------|
| **付费者=使用者** | 灌香肠加工、建材销售、本地服务 | 按本模板，直接需求+内容结构+上下游+配套 |
| **付费者≠使用者** | 奶粉、童装、礼品、养老服务 | 分离分类：使用者问题+付费者顾虑+产品推荐 |

> 💡 **本模板适用于：付费者=使用者的情况**
