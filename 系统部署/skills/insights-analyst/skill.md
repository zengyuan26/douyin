## 描述
这是一个用于市场洞察和数据分析的 skill。通过分析用户行为、消费习惯、市场趋势等数据，提供行业洞察和策略建议。支持两种客户类型：**卖货类**和**服务类**。

同时提供**社交媒体趋势监控接口**，支持抖音、快手、小红书、大众点评四大平台的全维度监控分析。

## 💡 核心思维：问题在先，人群在后

### ❌ 错误思路（先有人，再找需求）

```
先定义人群：新手妈妈、二胎妈妈
↓ 
找需求：担心假货、不会选
↓ 
卖产品：进口奶粉
```

### ✅ 正确思路（先有问题，再有人群）

```
先发现问题：孩子乳糖不耐受、便秘、过敏
↓ 
人群自动出现：有个这样的宝宝的妈妈
↓ 
提供解决方案：特殊奶粉 + 专业指导
```

### 市场洞察逻辑

```
问题 → 背后站的是人群 → 需要解决方案

孩子有问题 → 妈妈才需要买奶粉
水管漏水 → 业主才需要修水管
```

## 💡 进阶思维：产品服务化

> 💡 **核心理念**：卖产品不如卖服务，产品只是服务的载体

### ❌ 传统思路（单纯卖产品）

```
定位：我是卖XX产品的
思维：产品思维、交易思维
结果：陷入价格战、利润越来越薄
```

### ✅ 服务化思路（卖解决方案）

```
定位：我是提供XX解决方案/服务的
思维：服务思维、顾问思维、医生思维
结果：建立专业壁垒、难以被替代、定价权
```

### 产品服务化的三种形式

| 形式 | 说明 | 示例 |
|------|------|------|
| **顾问型** | 成为客户的咨询顾问 | 卖锁 → 安防顾问/开锁专家 |
| **医生型** | 像医生一样诊断+方案 | 卖电池 → 电池医生/电瓶诊疗 |
| **服务型** | 产品是服务的载体 | 卖水 → 企业饮水解决方案 |

### 行业分析时如何应用？

```
分析客户业务时，要帮助客户重新定位：

❌ 传统定位：他是卖XX产品的
✅ 新定位：他是提供XX解决方案/服务的

案例：
- 卖锁的 → 家庭安防顾问/开锁专家
- 卖电池的 → 电池医生/电瓶养护专家
- 卖水的 → 企业饮水解决方案专家
- 卖奶粉的 → 宝宝喂养顾问/育儿专家

在运营规划中体现：
- 账号定位：专家/顾问/人设
- 内容方向：知识分享、问题解决
- 转化方式：咨询+方案，而非单纯卖货
```

### 行业分析要点

| 分析维度 | 传统思维（红海） | 问题导向思维（蓝海） |
|----------|------------------|---------------------|
| 起点 | 人群标签 | 问题/痛点 |
| 需求挖掘 | 大众需求 | 功能性需求 |
| 机会发现 | 热门品类 | 细分问题 |
| 竞争策略 | 价格战 | 解决方案 |

### 分析框架

1. **先找问题** - 孩子有什么问题需要喝特殊奶粉？
2. **再算人群** - 有这个问题的宝宝有多少？
3. **后定策略** - 围绕问题提供解决方案
4. **避开竞争** - 问题导向，而非品类导向

### 🔥 新增：用户失败经历分析（高权重⭐）

> 💡 **核心洞察**：爆款内容往往来自用户真实的失败经历
> 
> **案例**：灌香肠爆款选题"为什么你灌的香肠一切就散，一晒就酸"（70万播放）
> - 这不是从"产品角度"想到的问题
> - 这是从"用户制作失败经历"中挖掘的问题

#### 失败经历挖掘维度

| 维度 | 说明 | 示例 |
|------|------|------|
| **制作失败类** | 用户自己制作时遇到的问题 | 灌的香肠一切就散、一晒就酸 |
| **使用失败类** | 用户使用产品时遇到的问题 | 香肠咬不动、太硬、發霉 |
| **选择失败类** | 用户选择时犯的错误 | 买错了品牌、买错了口味 |
| **储存失败类** | 用户储存不当导致的问题 | 香肠变质、有异味 |

#### 失败经历问题挖掘方法

```
Step 1: 问自己"用户在做这件事时，容易犯什么错？"
Step 2: 问自己"用户做成这件事，需要注意什么？"
Step 3: 问自己"用户在这个过程中，最容易失败在哪一步？"
Step 4: 问自己"用户失败后，会搜索什么问题？"

示例（灌香肠）：
- 容易犯什么错？→ 肥瘦比例不对、调料放错
- 需要注意什么？→ 晾晒时间、储存温度
- 最容易失败在哪一步？→ 切的时候散掉、晒完酸了
- 失败后搜索什么？→ 香肠一切就散怎么办、香肠发酸
```

#### 失败经历 → 选题转化公式

```
用户失败经历 = 爆款选题

公式：为什么你[做成某事]总是[失败]，3招教你解决

应用案例：
- 为什么你灌的香肠一切就散 → 制作失败
- 为什么你腌的腊肉一炒就硬 → 制作失败  
- 为什么你选的奶粉宝宝拉肚子 → 选择失败
- 为什么你买的家具没用多久就坏 → 使用失败
```

#### 1.1.2 长尾词挖掘四维度

```
🔍 长尾词挖掘维度

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【第一维度：问题导向型】

  挖掘用户真实问题，转化为精准长尾词

  • 用户问什么 → 转化为关键词
  • 痛点即流量 → 解决=转化

  示例：
  "灌香肠多少钱一斤" → 南漳灌香肠多少钱一斤
  "哪家好吃" → 南漳灌香肠哪家靠谱
  "卫生吗" → 南漳灌香肠卫生吗


【第二维度：场景细分型】

  按用户使用场景挖掘细分需求

  • 时间场景：年前、年后、旺季、淡季
  • 空间场景：县城、乡镇、上门、到店
  • 人物场景：送父母、送丈人、自己吃

  示例：
  南漳灌香肠上门服务
  南漳灌香肠礼盒送礼
  灌香肠配方自学


【第三维度：痛点规避型】

  挖掘用户担忧，提供解决方案

  • 怕踩坑 → 避坑指南
  • 怕被宰 → 价格透明
  • 怕不卫生 → 过程展示

  示例：
  南漳灌香肠避坑
  灌香肠怎么选不踩雷
  南漳灌香肠哪家靠谱


【第四维度：地域细分型】

  从大词切地域，获取本地精准流量

  • 省级→市级→县级→镇级
  • 本地+周边扩展

  示例：
  南漳灌香肠 → 襄阳市灌香肠 → 湖北灌香肠
  南漳县城灌香肠 → 南漳开发区灌香肠
```
