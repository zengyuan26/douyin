## 三、选题侧重点

### 3.1 选题公式

```
人设类选题 = 【身份】+ 【观点/态度】+ 【情感钩子】
```

| 元素 | 说明 | 示例 |
|------|------|------|
| 身份 | 你是谁+做了多久 | 72岁、22年、老师傅 |
| 观点 | 对某件事的态度 | 诚信比钱重要、吃亏是福 |
| 情感钩子 | 引发共鸣的点 | 傻、闲不住、认死理 |

### 2.2 选题方向矩阵

| 维度 | 选题方向 | 示例 | 目的 |
|------|----------|------|------|
| **对自己** | 坚持、专注、笨功夫 | 22年只做一件事 | 匠人感 |
| **对客户** | 诚信、信任、责任 | 老顾客吃惯了我不能不做 | 信任感 |
| **对同行** | 格局、态度、不争 | 不怕抢生意各做各的 | 格局感 |
| **对行业** | 看透、内幕、底线 | 这行水深得很 | 权威感 |
| **对钱** | 底线、原则、不骗 | 人不行东西再好也没用 | 价值观 |

### 2.3 人设选题核心原则

```
❌  不要讲产品功能
❌  不要讲制作技巧
❌  不要讲促销优惠

✅  要讲做人的道理
✅  要讲经历过的故事
✅  要讲真实的态度和观点
```

## 三、标题设计

### 3.1 标题公式

```
人设类标题 = 【身份标签】+ 【观点/态度】+ 【悬念/反差】
```

### 3.2 五种高点击标题模式

| 模式 | 公式 | 示例 |
|------|------|------|
| **身份开场型** | 【年龄/身份】+ 【做了什么事】+ 【感悟】 | 72岁做香肠22年，我总结出一句话 |
| **反常识型** | 【普遍认知】+ 【反转】+ 【原因】 | 有人说我傻，其实我看得最清楚 |
| **承诺型** | 【我XX年没做过的事】+ 【为什么】 | 我做了22年香肠，从来不干这种事 |
| **设问型** | 【别人常问的问题】+ 【我的回答】 | 总有人问我为什么卖这么贵 |
| **感慨型** | 【年龄】+ 【时间】+ 【人生感悟】 | 72岁了才明白，做人比做事难 |

### 3.3 标题检查清单

```
□ 是否有人物标签（年龄、身份、资历）
□ 是否有明确观点或态度
□ 是否引发好奇或共鸣
□ 是否口语化、生活化
□ 是否符合黄姐说话风格（朴实，真诚、不矫情）
```

## 四、内容结构（7张图）

### 4.1 七张图情绪曲线

```
情绪曲线图：
                    ↑
                   /│\ P3-P5
                  / │ \ 专注/获得感
                 /  │  \
                /   │   \
               /    │    \
P1 ↑↑↑      /     │     \        P6 温暖
期待/好奇   /      │      \      安心
           /       │       \
          /        │        \      P7 信任
         /         │         \    期待
        /          │          \
       /           │           \
      ─────────────┴───────────────→ 时间轴
     P1    P2    P3    P4    P5    P6    P7
```

### 4.2 每张图的核心任务

| 页面 | 情绪 | 核心任务 | 人设内容要点 |
|------|------|----------|--------------|
| **P1** | 期待/好奇 | 吸引点击 | 建立身份标签（72岁、22年） |
| **P2** | 焦虑/共鸣 | 挖掘痛点 | 讲行业/同行的不好（引发认同） |
| **P3** | 专注/信任 | 分析原因 | 为什么会有这些问题（展示洞察） |
| **P4** | 坚定/清晰 | 亮明态度 | 我是怎么做的/我的原则是什么 |
| **P5** | 确信/满足 | 案例佐证 | 用故事/客户反馈证明观点 |
| **P6** | 温暖/安心 | 情感升华 | 从做事上升到做人/人生感悟 |
| **P7** | 信任/期待 | 行动引导 | 品牌收尾+联系方式 |

### 4.3 每张图的内容公式

```
P1封面：身份 + 核心观点 + 悬念
    「72岁做香肠22年，我发现一个道理」

P2痛点：现象 + 情绪 + 共鸣
    「这些年，行业里让我寒心的事」

P3分析：原因 + 洞察 + 态度
    「为什么有人敢这样？因为算的是眼前账」

P4态度：原则 + 底线 + 承诺
    「我做香肠22年，守住三条底线」

P5佐证：故事 + 反馈 + 总结
    「有人说我傻，但老顾客们最清楚」

P6升华：人生 + 感悟 + 情怀
    「72岁了，不是闲不住，是责任」

P7收尾：品牌 + 引导 + 信任
    「22年老味道，灌香肠找黄姐」
```

## 七、叙事节奏

### 7.1 三段式节奏

```
┌─────────────────────────────────────────────────────────┐
│                     人设内容三段式                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  【第一段：抛问题】P1-P2                                │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                         │
│  用身份吸引关注，用痛点引发共鸣                          │
│  节奏：快-快                                            │
│                                                         │
│  【第二段：亮态度】P3-P5                                │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                       │
│  讲原因、讲原则、讲故事                                  │
│  节奏：慢-慢-快                                        │
│                                                         │
│  【第三段：升情感】P6-P7                                │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                         │
│  从做事升华到做人，品牌收尾                              │
│  节奏：慢-快                                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 7.2 节奏控制表

| 页面 | 内容密度 | 阅读时间 | 节奏 |
|------|----------|----------|------|
| P1 | 低（只放核心观点） | 3秒 | 快 |
| P2 | 中（3个痛点） | 5秒 | 快 |
| P3 | 中（2个分析） | 8秒 | 慢 |
| P4 | 高（3个原则） | 10秒 | 慢 |
| P5 | 中（故事为主） | 10秒 | 慢 |
| P6 | 低（情感为主） | 5秒 | 快 |
| P7 | 低（收尾为主） | 3秒 | 快 |

### 7.3 节奏要点

```
P1：快 - 不要啰嗦，直接亮观点
P2：快 - 痛点要准，一针见血
P3：慢 - 给用户思考时间
P4：慢 - 原则要清晰，一眼记住
P5：慢 - 故事要完整，有细节
P6：快 - 升华点到为止
P7：快 - 直接收尾不废话
```

## 十一、方法论总结

### 人设类内容核心公式

```
人设类内容 = 【真实身份】+ 【真诚态度】+ 【真情故事】
                   ↓              ↓              ↓
               72岁大姐       从不骗人        22年的故事
               22年资历       吃亏是福        老顾客的话
               朴实接地气     诚信为本        经历的事
```

### 一句话方法论

```
人设类内容不是说"我很好"
而是说"我是这样的人"
让用户因为认可这个人
而认可这个人的产品
```
