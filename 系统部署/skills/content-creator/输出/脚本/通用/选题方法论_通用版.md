## 二、人设类选题核心原则

### 2.1 核心原则：说事 > 说理

```
┌─────────────────────────────────────────────────────────┐
│                  人设内容核心原则                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ❌ 不要做                    ✅ 要做                   │
│  ─────────────────────────────────────────────          │
│  直接讲价值观                通过故事带出价值观         │
│  自问自答"我是怎样的人"      让用户自己得出结论         │
│  喊口号/贴标签              展示具体行为和选择         │
│  自说自话、王婆卖瓜          说事、聊他人、看选择       │
│  讲道理                     讲故事                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 人设内容三维框架

| 维度 | 做法 | 举例 |
|------|------|------|
| **对人** | 讲和某个人的故事 | 有个客户让我差点放弃... |
| **对事** | 讲处理一件具体事的逻辑 | 去年有个订单出问题，我这样处理的... |
| **对世界** | 讲对这个行业的看法 | 为什么这行越来越难做？ |
| **通过事** | 遇到什么事→怎么想→怎么做→结果 | 有个供应商要掺假，我拒绝了... |

### 2.3 核心公式

```
人设 = 故事 + 细节 + 你的选择
     = 不是我说的 → 是你感受到的

表达方式：展示（Show）> 声明（Tell）
```

### 2.4 人设类选题三大类型

| 类型 | 定义 | 核心目的 | 适合场景 |
|------|------|----------|----------|
| **价值观型** | 通过故事讲做人的道理 | 建立信任 | 品牌建设期 |
| **故事型** | 讲和人的故事 | 情感溢价 | 传播裂变期 |
| **观点型** | 讲看事的角度 | 建立权威 | 行业发声期 |

### 2.5 人设选题核心原则

```
┌─────────────────────┬─────────────────────┐
│      ❌ 不要做      │      ✅ 要做        │
├─────────────────────┼─────────────────────┤
│  讲产品功能/技巧    │  讲做人做事的道理   │
│  讲促销优惠活动     │  讲真实经历的故事   │
│  讲自己有多厉害     │  讲自己的态度观点   │
│  讲同行有多差       │  讲行业的见闻洞察   │
│  直接喊口号         │  通过故事带观点     │
│  自说自话           │  让用户自己得出结论 │
└─────────────────────┴─────────────────────┘
```

## 三、选题侧重点

### 3.1 选题公式

```
人设类选题 = 【身份标签】+ 【故事/事件】+ 【你的选择】
```

| 元素 | 说明 | 示例 |
|------|------|------|
| 身份标签 | 你是谁+做了多久+有什么特点 | 年龄/姓氏/资历/手艺/性格 |
| 故事/事件 | 讲具体的事（对人对事对世界） | 某个客户的故事/处理某件事的过程 |
| 你的选择 | 遇到这件事你怎么想、怎么做 | 我的选择/我的底线/我的态度 |

### 3.2 选题方向矩阵

| 维度 | 选题方向 | 内容角度 | 目的效果 |
|------|----------|----------|----------|
| **对自己** | 坚持、专注、笨功夫 | 这么多年只做一件事 | 匠人感 |
| **对客户** | 诚信、信任、责任 | 为什么老客户一直认我 | 信任感 |
| **对同行** | 格局、态度、不争 | 不怕竞争，各做各的 | 格局感 |
| **对行业** | 看透、内幕、底线 | 这个行业的门道 | 权威感 |
| **对钱** | 底线、原则、不骗 | 有些钱不能赚 | 价值观 |
| **对人生** | 感悟、态度、情怀 | 做这行教会我什么 | 温度感 |

## 四、标题设计

### 4.1 标题公式

```
人设类标题 = 【身份标签】+ 【观点/态度】+ 【悬念/反差】
```

### 4.2 五种高点击标题模式

| 模式 | 公式 | 示例 |
|------|------|------|
| **身份开场型** | 【年龄/身份/资历】+ 【核心观点】 | 做了20年XX，我总结出一句话 |
| **反常识型** | 【普遍认知】+ 【反转观点】 | 有人说我傻，其实我看得最清楚 |
| **承诺型** | 【我做XX年，从不XX】 | 做这行15年，从来不干这种事 |
| **设问型** | 【别人常问的问题】+ 【我的回答】 | 总有人问我为什么卖这么贵 |
| **感慨型** | 【年龄/时间】+ 【人生感悟】 | XX岁了才明白，做人比做事难 |

### 4.3 标题检查清单

```
□  是否有人物标签（年龄/身份/资历/姓氏）
□  是否有明确观点或态度
□  是否引发好奇或共鸣
□  是否口语化、生活化
□  是否符合人设说话风格（朴实/专业/亲切）
□  是否控制在20字以内（小红书）/ 30字以内（抖音）
```

### 4.4 标题情绪词库

| 情绪类型 | 适用词 | 效果 |
|----------|--------|------|
| 真诚型 | 心里话、说实话、老实说 | 建立信任 |
| 感慨型 | 这么多年、想起来、悟出一个理 | 引发共鸣 |
| 坚定型 | 从来不、我一直坚信、绝对不 | 强化态度 |
| 反问型 | 你们说呢、是不是、认不认同 | 引发互动 |

## 六、情绪设计

### 6.1 情绪类型与适用场景

| 情绪类型 | 适用场景 | 示例文案 | 人设风格适配 |
|----------|----------|----------|--------------|
| **真诚** | 所有页面 | 说句心里话、跟你们讲讲 | 所有类型 |
| **感慨** | P1/P6 | 这么多年、想起来真是 | 老师傅/匠人 |
| **坚定** | P4/P5 | 从来不、我一直坚信 | 老板娘/创始人 |
| **温暖** | P5/P6 | 谢谢大家、老客户们 | 亲和型人设 |
| **幽默** | P2/P5 | 傻人有傻福、被人说傻 | 性格开朗型 |
| **专业** | P3/P4 | 从专业角度讲、从数据看 | 专家型人设 |

### 6.2 情绪递进表

```

### 12.2 一句话方法论

```
人设类内容不是说"我很好"
而是说"我是这样的人"
让用户因为认可这个人
而认可这个人的产品
```

```
┌─────────────────────────────────────────────────────────┐

### 15.1 问题诊断类

#### 15.1.1 核心原则：痛点 > 解决方案

```
┌─────────────────────────────────────────────────────────┐
│                  问题诊断类核心原则                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ❌ 不要做                    ✅ 要做                   │
│  ─────────────────────────────────────────────          │
│  直接给答案                  先挖掘痛点引发共鸣         │
│  泛泛而谈"有问题"           具体到某个症状/场景         │
│  标题太温和                  标题要戳中要害，让人刺痛    │
│  只讲问题不共情              讲问题的同时让用户感到被理解│
│  解决方案太早抛出            先共情，再给方案            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 15.1.2 选题三维框架

| 维度 | 做法 | 举例 |
|------|------|------|
| **对症状** | 讲具体的表现/频率 | 宝宝揉眼睛揉到红，哭闹不止 |
| **对原因** | 讲为什么会这样 | 可能是这三个原因导致的 |
| **对后果** | 讲不处理的代价 | 长期不管会影响XX |
| **对解决** | 讲正确的方向 | 关键是要做到这3点 |

#### 15.1.3 选题三大类型

| 类型 | 定义 | 核心目的 | 适合场景 |
|------|------|----------|----------|
| **症状罗列型** | 讲清楚有哪些症状表现 | 引发对照 | 初期涨粉 |
| **原因分析型** | 深挖为什么会出问题 | 建立专业感 | 建立权威 |
| **后果警示型** | 讲不处理的严重后果 | 制造紧迫 | 推动转化 |

#### 15.1.4 选题来源挖掘

```
┌─────────────────────────────────────────────────────────┐

#### 15.5.1 核心原则：易懂 > 专业

```
┌─────────────────────────────────────────────────────────┐

#### 15.5.2 选题三维框架

| 维度 | 做法 | 举例 |

| **对原理** | 讲明白为什么 | 因为缺少乳糖酶，消化不了奶粉 |

#### 15.5.3 选题三大类型

| 类型 | 定义 | 核心目的 | 适合场景 |
|------|------|----------|----------|

### 15.8 避坑指南类

#### 15.8.1 核心原则：警示 > 提醒

```

│                                                         │

│  ─────────────────────────────────────────────          │

│  讲完就结束                    要给出正确做法            │

#### 15.8.2 选题三维框架

```
