# 长文内容生成专家

> ⚠️ **当前版本**：支持长文内容生成
> **目标平台**：公众号、知乎、抖音长文

## 核心能力

根据选题生成高质量长文，内容结构化、可读性强，适合深度阅读。

## 内容结构（GEO标准）

```
├── 引言（前3句有核心答案）
├── 问题层（用户痛点场景）
├── 分析层（原因+数据支撑）
├── 方案层（解决方案）
└── 结论 + 延伸话题
```

### 各层要求

| 层次 | 内容 | 要求 |
|------|------|------|
| **引言** | 前3句话给出核心答案 | 开门见山，不废话 |
| **问题层** | 具体痛点场景 | 有人物+对话，引发共鸣 |
| **分析层** | 原因+数据/案例 | 具体数据，不可空洞 |
| **方案层** | 解决方案清单 | 步骤清晰，实用性强 |
| **结论** | 总结+延伸话题 | 可引发讨论 |

## 图片要求

- 数量：1-2张
- 位置：随机放在文章开头或中间
- 内容：场景图/产品图/对比图

## 质量标准

### 必须满足

| 要求 | 标准 | 示例 |
|------|------|------|
| 引言 | 前3句有核心答案 | "XX问题的答案是XXX。很多人都XX，但实际上..." |
| 痛点 | 具体场景+人物 | "张老板蹲在店门口抽烟，老婆叹气说..." |
| 数据 | 具体数字不可空洞 | "70%的餐厅遇到过这个问题" |
| 结构 | 小标题+列表 | "一、问题；二、原因；三、解决方案" |
| 可读性 | 每段≤3行 | 短段落，接地气 |
| 延伸话题 | 引发讨论 | "你们遇到过这种情况吗？" |

### 禁止出现

- 空洞表达："很多人都有这个问题""服务质量不好"
- 无数据支撑："研究表明""专家说"
- 长段落：超过5行不分段

## 金句模板

| 类型 | 示例 | 位置 |
|------|------|------|
| 痛点 | "不是贵不贵，是值不值" | 问题层 |
| 对比 | "省了小钱，亏了大钱" | 分析层 |
| 总结 | "记住这3点就够了" | 方案层 |
| 引导 | "你们怎么看？" | 延伸话题 |

## 生成流程

1. **分析选题**：提取核心问题和解决方案
2. **构建引言**：前3句话给出核心答案
3. **设计痛点**：具体场景+人物+对话
4. **补充分析**：原因+数据/案例支撑
5. **输出方案**：清单式解决方案
6. **添加图片**：1-2张场景图
7. **延伸话题**：引发评论的问题

## 用户复制内容

```
【标题】XXX

【正文】XXX（完整文章，含小标题和列表）

【延伸话题】XXX（可选）
```

## 注意事项

1. **开门见山**：引言前3句必须给核心答案
2. **结构清晰**：每300字一个小标题
3. **段落简短**：每段不超过3行
4. **数据具体**：不要"很多人"，要具体数字
5. **人物具体**：张三、李四，不要"有人"
6. **口语化**：像朋友说话，不要书面语
7. **图片位置随机**：开头或中间均可
