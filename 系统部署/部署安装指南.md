# AI棱镜内容营销系统 - 部署安装指南

> **系统名称**：AI棱镜
> **版本**：v3.3.2
> **技术栈**：Flask + SQLite + Bootstrap + LLM（硅基流动/ollama）

---

## 一、系统概述

### 1.1 产品定位

**AI棱镜** 是一套基于 Master-Worker 架构的**多平台内容营销智能系统**。采用"对话+工具"的混合模式，支持客户资料收集、市场分析、SEO 优化、内容创作等全链路营销工作。

### 1.2 核心功能

| 模块 | 功能 |
|------|------|
| 画像系统 | 用户画像生成与管理、关键词库、选题库 |
| 内容生成 | 图文脚本、短视频脚本、长文本生成 |
| SEO优化 | 关键词挖掘、选题规划、标签优化 |
| 运营规划 | 账号设计、人设定位、变现路径 |
| 专家系统 | 8大AI专家协作（首席营销官、市场洞察、Geo SEO、运营、内容等） |

### 1.3 系统架构

```
用户浏览器
    ↓
Flask Web服务（端口5001）
    ↓
SQLite数据库
    ↓
LLM服务（硅基流动API / Ollama本地）
```

## 三、部署步骤

### Step 1：下载源码

```bash
# 方式1：直接复制整个系统部署目录
cp -r /Volumes/增元/项目/douyin/系统部署 /目标路径/

# 方式2：如果是打包文件，解压到目标路径
unzip douyin_system.zip -d /目标路径/
```

### Step 2：创建虚拟环境（推荐）

```bash
cd /目标路径/系统部署

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
# macOS/Linux:
source venv/bin/activate

# Windows:
venv\Scripts\activate
```

### Step 3：安装依赖

```bash
# 激活虚拟环境后执行
pip install -r requirements.txt
```

**requirements.txt 内容**：

```txt
Flask==2.3.3
Flask-SQLAlchemy==3.1.1
Flask-Login==0.6.3
Flask-Bcrypt==1.0.1
Werkzeug==2.3.7
python-dotenv==1.0.0
gunicorn==21.2.0
requests==2.31.0
APScheduler==3.10.4
PyPDF2==3.0.0
ebooklib==0.18
```

### Step 4：配置环境变量

```bash
# 创建 .env 文件
cp .env.example .env
```

编辑 `.env` 文件，配置以下内容：

```bash
# ========== LLM配置（必须）==========

# 方式A：使用硅基流动（推荐，云端服务）
LLM_PROVIDER=siliconflow
LLM_MODEL=Qwen/Qwen2.5-14B-Instruct
LLM_BASE_URL=https://api.siliconflow.cn/v1
LLM_API_KEY=你的API密钥

# 方式B：使用Ollama（本地服务）
# LLM_PROVIDER=ollama
# LLM_MODEL=qwen2.5:7b
# LLM_BASE_URL=http://localhost:11434
# LLM_API_KEY=

# ========== Flask配置 ==========
SECRET_KEY=你的随机密钥（生产环境必改）
FLASK_ENV=development
FLASK_DEBUG=1

# ========== 数据库配置 ==========
# 默认使用SQLite（开发环境）
DATABASE_URL=sqlite:///instance/douyin_system.db
```

### Step 5：初始化数据库

```bash
# 首次运行，系统会自动创建数据库
# 如需手动初始化：
python3 -c "from database.init_db import init_database; init_database()"
```

### Step 6：创建管理员账号

```bash
# 默认管理员（已预置）
用户名：admin
密码：admin123

# 如需重置密码：
python3 reset_admin_password.py
```

## 四、启动系统

### 4.1 开发环境启动

```bash
cd /目标路径/系统部署
source venv/bin/activate

# 方式1：直接运行
python3 app.py

# 方式2：带参数运行
python3 app.py --host=0.0.0.0 --port=5001 --debug
```

### 4.2 生产环境启动

```bash
cd /目标路径/系统部署
source venv/bin/activate

# 使用gunicorn
gunicorn -w 4 -b 0.0.0.0:5001 app:app
```

### 4.3 后台运行（推荐）

```bash
# 使用nohup后台运行
nohup python3 app.py > nohup.out 2>&1 &

# 查看进程
ps aux | grep app.py

# 查看日志
tail -f nohup.out
```

### 4.4 访问系统

启动后，在浏览器访问：

| 环境 | 地址 |
|------|------|
| 本地 | http://127.0.0.1:5001 |
| 局域网 | http://你的IP:5001 |
| 公网 | http://你的域名:5001 |

## 六、LLM服务配置

### 6.1 硅基流动（推荐）

1. 注册硅基流动账号：https://siliconflow.cn
2. 获取API Key
3. 在 `.env` 中配置：

```bash
LLM_PROVIDER=siliconflow
LLM_MODEL=Qwen/Qwen2.5-14B-Instruct
LLM_BASE_URL=https://api.siliconflow.cn/v1
LLM_API_KEY=sk-xxxxxxxxxxxxxxxx
```

### 6.2 Ollama（本地）

1. 安装Ollama：https://ollama.ai
2. 下载模型：

```bash
# 下载千问模型
ollama pull qwen2.5:7b

# 或下载更大模型
ollama pull qwen2.5:14b
```

3. 在 `.env` 中配置：

```bash
LLM_PROVIDER=ollama
LLM_MODEL=qwen2.5:7b
LLM_BASE_URL=http://localhost:11434
LLM_API_KEY=
```

4. 启动Ollama服务：

```bash
ollama serve
```

## 八、常见问题

### Q1：启动报错 "ModuleNotFoundError"

```bash
# 确保已激活虚拟环境并安装依赖
source venv/bin/activate
pip install -r requirements.txt
```

### Q2：LLM调用失败

```bash
# 检查API配置
cat .env | grep LLM

# 测试API连通性
curl -X POST https://api.siliconflow.cn/v1/chat/completions \
  -H "Authorization: Bearer 你的API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"Qwen/Qwen2.5-14B-Instruct","messages":[{"role":"user","content":"Hello"}]}'
```

### Q3：端口被占用

```bash
# 查找占用端口的进程
lsof -i :5001

# 杀掉进程
kill -9 进程PID

# 或使用其他端口
python3 app.py --port=5002
```

### Q4：数据库锁定

```bash
# 检查数据库文件权限
ls -la instance/douyin_system.db

# 修复权限
chmod 644 instance/douyin_system.db
chmod 755 instance/
```

## 九、生产环境部署

### 9.1 Nginx反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 9.2 HTTPS配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:5001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 9.3 Systemd服务

创建 `/etc/systemd/system/aiprism.service`：

```ini
[Unit]
Description=AI Prism Content Marketing System
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/系统部署
ExecStart=/path/to/系统部署/venv/bin/python3 app.py
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable aiprism
sudo systemctl start aiprism
```
