# 图文内容模板

> ⚠️ **【强制规格】内容形式：图文 | 图片数量：6-7张 | 比例：9:16竖版 1080×1920px**
>
> 💡 **版本：2.0.0 | 新增：两阶段流水线（导演分析→逐页生成）+ 连续性系统（transition/carryOver/newElements/emotionLink）**
>
> ⚠️ **生成流程**：Step 8a 先出导演分析（主线/情绪曲线/观众追踪/视觉比喻），用户确认后再 Step 8b 逐页生成。**禁止跳过 Phase 1 直接出全文。**

## 📐 爆款公式速查

```
盘点类爆款 = 数字 + 对比/排名 + 强烈观点 + 时效性

标题 = 数字（3个/5款）+ 对比（vs/比谁强）+ 悬念（买了后悔/只有2个能用）+ 时效（2026/最新）
结构 = 强钩子(1屏) → 核心结论(1屏，用对比表) → 逐条拆解(3-4屏，配图+个人体验) → 互动收尾(1屏)
争议 = 全文只放1个争议点，不要超过1个
文字 = 每页≤60字，用表格/对比图代替段落
```

## 二、标题设计

> ⚠️ **标题决定80%的点击率。必须包含以下4个要素中的至少3个。**

### 2.1 爆款标题公式

```
爆款标题 = 数字 + 对比/排名 + 强烈观点 + 时效性
```

| 技巧 | 说明 | 示例 |
|------|------|------|
| **加数字** | 3个、5款、10大 | "3个买了就后悔的配件" |
| **加对比/排名** | vs、比、谁更强、横评 | "5款AI编程助手横评，只有2个能用" |
| **加悬念/反转** | 买了后悔、别急着买、没想到 | "我测了5款，这款让我坚持了30天" |
| **加时效性** | 2026年、最新、刚闭幕 | "SNEC 2026刚闭幕，3个变化" |

### 2.2 标题对比

```
❌ 差：MacBook配件推荐
✅ 好：2026年MacBook配件盘点，这3个买了就后悔

❌ 差：储能行业趋势分析
✅ 好：SNEC 2026刚闭幕，储能3个变化，跟工厂电费直接相关

❌ 差：健身app推荐
✅ 好：10款健身app深度体验，这款让我坚持了30天
```

### 2.3 主标题

【主标题——必须包含数字+观点】

### 2.4 副标题

【副标题——建议用vs对比结构】

### 2.5 封面建议

| 要素 | 要求 |
|------|------|
| **开头词** | 数字/时效/身份标签 |
| **情绪驱动** | 损失厌恶优先："再不…就…""差距""多交了XX" |
| **行动引导** | 第X个最关键→ / 看到最后→ |
| **必须包含** | 数字背书 + 具体数据（不要模糊的"几十万"，要具体的"35万"） |

## 三、内容规划（黄金结构）

> ⚠️ **使用"黄金结构"替代旧的情绪动线七页模型。核心原则：前3屏定生死。**

### 黄金结构速查表

| 页面 | 定位 | 核心任务 | 形式要求 |
|------|------|----------|----------|
| P1 | **强钩子** | 一张图+一句话，让用户停下滑动 | 信息密度高，一眼看懂 |
| P2 | **核心结论** | 直接给答案，不要铺垫 | **用对比表，不要大段文字** |
| P3～N-1 | **逐条拆解** | 每条配图，加个人体验 | 卡片式，每页≤60字 |
| P末 | **互动收尾** | 提问/投票/CTA，引导评论或私信 | 路径清晰，成本低 |

### 页面数量

```
盘点/干货类：6张（P1封面 → P2结论 → P3-P5拆解 → P6 CTA）
问题解决类：7张（P1封面 → P2痛点 → P3-P5方案 → P6升华 → P7 CTA）
```

### 连续性系统（P2 起每页强制）

> 从 zimeiti-creator 分镜连续性迁移。每页不是孤岛，是连续叙事的一个节拍。

每页（P2-P7）必须标注以下 4 个连续性字段：

| 字段 | 含义 | 示例 |
|------|------|------|
| **transition** | 翻页衔接方式 | 硬切 / 叠化 / 色块过渡 / 意象延续 |
| **carryOver** | 上一页哪个元素保留 | 琥珀色条 / 沙漏图标 / "信息差"关键词 / 冷灰底色 |
| **newElements** | 这一页新增什么信息 | 对比表 / 数据卡片 / 争议点 |
| **emotionLink** | 情绪变化（上一页→本页） | 好奇→恍然 / 乐观→清醒 / 清醒→确信 |

**硬性要求**：
- P2 连续性指向 P1（翻页衔接 + 情绪变化）
- 关键视觉隐喻（如沙漏/天平）在 ≥3 页的 carryOver 中出现
- P4→P5 的 emotionLink 是整篇最关键的情绪转折，必须写清楚过渡方式

### P1：封面 — 强钩子

**【画面描述】**
背景：暗调场景虚化（根据选题定制）
布局：竖版9:16，暗角聚焦中心文字
光影：冷调或暗调（制造紧迫感），损失厌恶类内容优先用冷暗调
人物：可出镜可不出镜，不出镜时用场景图
禁止：手机Mockup、emoji

**【文字内容】**
```
主标题：【一句话钩子，包含数字或对比。例：SNEC 2026刚闭幕。一样的工厂，隔壁一年电费比你少交35万】
金句块：【制造损失感或认知差距。例：3个信号——差距不是电价，是储能】
行动引导：【第X个最关键→ / 看到最后→】
副标题：【价值承诺。例：长时储能落地 · 钠电交付 · 基金成立】
```

**封面情绪技巧**：
- 损失厌恶优先："再不…就…""比你少交XX""差距正在拉开"
- 避免空洞的"必看""揭秘""大错特错"——这些已被用烂，没有信息量

### P2：核心结论

> ⚠️ **P2是整篇的灵魂。必须直接给答案，禁止铺垫、禁止大段文字。**

**【连续性】**
```
P1→P2:
  transition: 【翻页衔接方式】
  carryOver: 【从封面延续的元素，如：冷灰底色/标题关键词/问号意象】
  newElements: 【本页新增的信息，如：对比表/核心结论/数据卡片】
  emotionLink: 好奇 → 恍然
```

**【画面描述】**
背景：冷灰渐变暖色（从问题过渡到答案）
布局：竖版9:16，对比表/数据卡片 + 一句结论
元素：用表格、前后对比、左右对照呈现，不要段落
人物：按需
禁止：手机Mockup、emoji、超过3行的文字段落

**【文字内容】**
```
主标题：【结论性判断，一句话。例：这次展会看完，储能从可选项变成了必选项】

对比表：
去年 | 今年
【状态A1】 | 【状态B1】
【状态A2】 | 【状态B2】
【状态A3】 | 【状态B3】

底部结论：【不超过20字。例：之前叫讲故事，之后叫拼交付】
```

---

### P3～P5：逐条拆解

> ⚠️ **每页拆一个点。配图 + 数据对比 + 个人体验。每页≤60字。**

**【连续性】（每页独立填写）**
```
P2→P3:
  transition: 【翻页衔接方式】
  carryOver: 【从 P2 延续的元素】
  newElements: 【本页新增的信息】
  emotionLink: 恍然 → 专注

P3→P4:
  transition: 【翻页衔接方式】
  carryOver: 【从 P3 延续的元素】
  newElements: 【本页新增的信息】
  emotionLink: 专注 → 乐观

P4→P5:
  transition: 【翻页衔接方式。⚠️ 全篇最关键转折，过渡不能太生硬】
  carryOver: 【从 P4 延续的元素】
  newElements: 【本页新增的信息】
  emotionLink: 乐观 → 清醒  ⚠️ 必须用一句话过渡，不超过 12 字
```

**【画面描述】**
背景：逐页升温（冷灰→米色→暖白）
布局：竖版9:16，徽章编号 + 关键数据 + 个人感受区
元素：图标/实物图 + 对比数据卡 + "我在现场/我的感受"标签
人物：按需
禁止：手机Mockup、emoji

**【文字内容】**
```
主标题：趋势① / 要点① 【名称】

对比数据：【用对比呈现，不要平铺。例：以前2-4小时 → 现在8小时现货】

个人体验：【一句话，增加真实感。例：去年还在展板上，今年伸手就能摸到机器】

一句话观点：【不超过15字。例：两年前装的2小时系统，可能要过时了】
```

**争议点规则**：
- 全文**只放1个争议点**，不要超过1个
- 放到最合适的拆解页里（通常是P4）
- 争议点结尾带提问："你怎么看？"

---

### P末：互动收尾

**【连续性】**
```
P(N-1)→P末:
  transition: 【翻页衔接方式】
  carryOver: 【从上一页延续的元素，如：品牌色/核心比喻/情绪基调】
  newElements: 【本页新增：CTA/金句/品牌标识】
  emotionLink: 清醒 → 确信 / 确信 → 判断  ⚠️ 收尾不能停在焦虑，必须给出路
```

**【画面描述】**
背景：品牌主色
布局：竖版9:16，中央大字 + CTA
元素：品牌logo + 行动按钮
人物：按需
禁止：手机Mockup、emoji

**【文字内容】**
```
主标题：【行动号召，跟前面内容呼应】

金句块：【降低行动门槛。例：免费评估 · 不是推销 · 算清楚再说】

行动按钮：【简短动词短语。例：私信"储能"】

品牌标识：品牌名 | 业务关键词
```

## 四、盘点类图文专项规范

> 盘点类图文是**最容易出爆款**的内容形式。核心逻辑：帮用户节省时间、降低认知成本。

### 4.1 盘点选题类型

| 类型 | 爆款逻辑 | 举例 |
|------|---------|------|
| 刚需盘点 | 用户主动搜索 | "2026年最值得买的10款 Mac 配件" |
| 争议盘点 | 引发站队评论 | "5款主流 AI 编程助手横评，谁在裸泳" |
| 时效盘点 | 蹭热点流量 | "WWDC 2026 最值得关注的7个更新" |

### 4.2 盘点类特殊要求

```
✅ 每页必须有对比、排名或评分，不能只罗列
✅ P2核心结论用对比表，不要写"为什么做这个盘点"
✅ P3-P5每条 = 1个趋势/产品/观点 + 1个数据 + 1句个人判断
✅ 全文1个争议点，放在中间位置（通常P4）
✅ 标题必须包含：数字 + 时效词 + 强烈观点
❌ 禁止平铺罗列，禁止没有个人判断的中性介绍
❌ 禁止超过1个争议点，多了用户记不住也不愿意评论
```

## 六、制作规范

### 设计规格

```
✅ 图片数量：6-7张（9:16竖版）
✅ 文字大小：标题≥40pt，金句≥40pt，正文≥16pt
✅ 配色：统一色系，不超过3种主色
✅ 数据可视化：能用对比图的不用文字描述（左右对比 > 单独介绍）
✅ 核心观点：每张图有1个金句视觉块
✅ 案例数据：必须具体（具体数字>模糊说法。例："35万"不是"几十万"）
✅ 互动引导：末尾必须有CTA或提问
✅ 人物：可出镜可不出镜，统一即可
✅ 文字密度：每页≤60字，手机端可读性优先
✅ 禁止项：手机Mockup、emoji、特殊符号、纯色/渐变背景、大段文字段落（超过3行）
```

### 字体规范

```
✅ 字体选择（只使用这4种）：
   思源黑体（推荐）、微软雅黑、华文细黑、Arial

✅ 禁止特殊字符：
   代码块符号、数学符号、装饰符号、Emoji

✅ 中文标点（只使用）：
   逗号（，）、句号（。）、顿号（、）、冒号（：）、引号（「」"）

✅ 排版简化：
   文字尽量一行说完
   每张图≤60字
   优先用对比表代替段落
   禁止竖排文字
```

## 七、留存优化

### 7.1 系列化内容（最重要）

```
不要：发一篇独立的盘点
要做：做成系列 IP

例：
  Vol.1 — SNEC 2026储能展3个变化
  Vol.2 — 虚拟电厂接入全流程
  Vol.3 — 工厂省电案例实录
  → 用户会主动点进主页看全集
```

### 7.2 内容内埋钩子

```
中间埋钩子：
  "这个点争议最大，你怎么看？评论区说"

结尾埋钩子：
  "下期盘点XX，关注不迷路"
  "你觉得还有哪些遗漏？评论区告诉我"
  "点赞过1000，出详细拆解"
```

### 7.3 收藏价值优化

> 用户收藏 = 内容对他有长期价值

**高收藏率内容特征：**
- 有对比表（可以截图保存）
- 有评分/排名（可以做决策参考）
- 有具体数据（可以引用）
