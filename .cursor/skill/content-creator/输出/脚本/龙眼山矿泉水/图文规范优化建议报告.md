# 现有 skill.md 图文设计方法论优化建议报告

> 对比分析：现有 v1.7.0 社交媒体长图文设计方法论 vs 新版「社交媒体多图短视频·纯画面版内容设计方法论」
>
> 分析日期：2026-04-28
>
> 核心结论：两套方法论框架高度重合，底层逻辑一致。现有 skill 的四维设计系统（D1-D4）已经涵盖了新方法论的核心要素。但**新方法论在执行细节上更严格、更画面化**，部分术语和执行指令需要吸收整合。

---

## 一、整体对比总览

### 1.1 框架重合度分析

| 现有 skill 维度 | 新方法论维度 | 重合度 | 差异点 |
|---|---|---|---|
| D1 情绪动线逻辑 | 维度一：情绪动线逻辑 | **高度重合** | 新方法论引入了「镜头」概念和「画面叙事流」，更强调纯视觉引导 |
| D2 视觉一致性算法 | 维度二：视觉资产一致性算法 | **高度重合** | 新方法论更精细化地规定了色调冷暖与情绪阶段的对应关系 |
| D3 信息层级架构 | 维度三：纯画面信息层级架构 | **中度重合** | 新方法论更激进——要求「零密集文字、轻量化视觉符号」，禁用大段段落文字 |
| D4 高转化排版模板 | 维度四：高转化纯画面版式 | **高度重合** | 4套版式命名不同（引流/痛点/干货/收尾 vs billboard/problem-solver/matrix/trust-builder），本质相同 |
| 系统执行指令 | 无对应章节 | **新增** | 新方法论末尾有可直接喂给 AI/生图工具的系统指令 |

### 1.2 核心差异一句话总结

> 现有 skill 是**「图文设计系统」**（图文笔记为主），新方法论是**「纯画面叙事系统」**（短视频画面为主）。两者情绪框架一致，但**新方法论的信息密度更低、画面叙事权重更高、文字约束更严格**。

## 二、逐维度详细差异分析

### 2.1 维度一：情绪动线逻辑

**现有 D1 vs 新维度一：完全对应**

| 现有 skill | 新方法论 | 对比 |
|---|---|---|
| P1 封面「期待/好奇」 | 镜头1【引流钩子画面】「制造好奇、勾起期待」 | ✅ 一致 |
| P2 痛点「焦虑/共鸣」 | 镜头2【共情痛点画面】「戳中焦虑、建立共鸣」 | ✅ 一致 |
| P3-P5 获得感 | 镜头3-5【价值干货画面】「输出专业、给到解决方案」 | ✅ 现有拆成3页，新方法论合并为「3-5」，本质相同 |
| P6 升华「温暖/安心」 | 镜头6【治愈升华画面】「舒缓情绪、传递温暖安心」 | ✅ 一致 |
| P7 转化「信任/期待」 | 镜头7【信任转化画面】「强化认可、引导行动沉淀」 | ✅ 一致 |

**吸收建议：**
- 新方法论的镜头命名更有画面感（引流→共情→干货→治愈→信任），可直接引入替代现有 P1-P7 命名
- 新方法论明确了「整套短视频固定7帧画面镜头」，强化了约束性

**关键差异——内容页（P3-P5）拆分逻辑：**

现有 skill：P3分析 → P4方案 → P5落地（三步递进）
新方法论：镜头3/4/5【价值干货画面】「分镜拆分展示方法、场景化分步演示、对比式画面呈现」

差异不大，但新方法论更强调「分镜思维」而非「步骤递进」。建议：**在 D1 中引入「分镜叙事」概念**，让内容页更有画面感。

### 2.2 维度二：视觉一致性算法

**现有 D2 vs 新维度二：高度重合，执行细节更精细**

| 现有 D2 子项 | 新维度二 子项 | 对比 |
|---|---|---|
| 2.1 人物连贯性 | 人物/主体连贯性 | ✅ 完全一致 |
| 2.2 光影逻辑（High-key柔和光） | 光影镜头逻辑（柔和自然光、低饱和柔焦） | ✅ 核心一致，术语更精准 |
| 2.3 色彩递进算法 | 色调动态递进 | ✅ 完全一致 |

**关键差异——色调递进与情绪的联动：**

新方法论的色调动态递进更精细：

| 页面 | 现有 skill 色值 | 新方法论色值 | 新方法论的「禁用」约束 |
|---|---|---|---|
| 痛点镜头 | 冷灰色系 #708090/#A9A9A9 | 低饱和灰调、冷雾感 | 强化压抑痛点氛围 |
| 干货镜头 | 暖色系 #F5F5DC/#FFE4C4 | 自然暖调、柔光浅色系 | 凸显舒适、专业、治愈感 |
| 收尾转化 | 品牌主色 | 品牌主色小面积点缀 | 做视觉闭环记忆点 |

**吸收建议：**
- 现有 skill 的色值参考已经很精确，新方法论的表述可以作为补充说明（帮助 AI 更准确理解色彩意图）
- 新方法论强调「冷雾感」「柔光浅色系」等视觉描述词汇，建议在 AI 生图约束中加入这些关键词

### 2.3 维度三：信息层级架构（最大差异）

**现有 D3 vs 新维度三：理念相同，执行约束差异显著**

| 现有 D3 | 新维度三 | 差异分析 |
|---|---|---|
| 「1-3-5」扫描原则（1个金句+3个模块+5秒闭环） | 「1核·3区·5秒看懂」纯视觉原则 | 本质相同，新方法论更强调「纯视觉符号」 |
| UI元素化（气泡框、勾选框、时间轴等） | 视觉化：图标符号、色块分区、对比构图 | 新方法论更简洁，但缺少 UI 组件的具体类型指引 |
| 金句视觉块（每张1个，≥40pt） | 极简短句标签、关键词小字点缀 | **核心差异**：新方法论禁用大段段落文字，只保留极简关键词 |

**最关键差异：文字密度约束**

| 对比项 | 现有 skill | 新方法论 |
|---|---|---|
| 金句块要求 | ≥40pt，带背景色块（占画面5-10%） | 无大段文字，只保留「极简短句标签、关键词小字点缀」 |
| 干货呈现方式 | 分步拆解 + UI组件（气泡框/时间轴/对比图） | 分镜、对比、特写、分步场景镜头可视化 |
| 禁用项 | emoji、特殊符号 | **大段段落文字**（这是最严格的约束） |

**吸收建议：**
- 这是最需要吸收的差异。新方法论对「纯画面叙事」的坚持更彻底
- 建议在 D3 中新增一条硬约束：**P3-P5（干货页）禁用大段段落文字，干货内容全部画面化呈现**
- 但保留现有 skill 的 UI 组件体系（气泡框、勾选框等），因为图文笔记场景确实需要这些元素

### 2.5 系统执行指令（新增章节）

新方法论末尾有一节「系统通用执行指令」，可以直接喂给 AI/生图工具：

```

- 这个章节对 AI 生图工具特别有价值
