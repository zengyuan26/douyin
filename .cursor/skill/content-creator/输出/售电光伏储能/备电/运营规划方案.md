## 一、核心定位

### 一句话

**CVC投行投资人。看新能源、看AI、看人看项目。AI让信息平权了，但散落的信息拼成框架、给出判断——这个能力AI替代不了。**

不是教工厂老板怎么做工厂。不是讲投资知识。不是AI教程。是**把你看到的信息拼成判断，让别人拿走就能用**。

不招居间人。不做居间人。目标客户是有资产的人——工厂老板、企业主、高净值个人。

### 为什么是你

| 层 | 能力 | 观众看到什么 |
|------|------|------|
| 实战判断 | 多次创业+多家公司GM。成过败过，一眼看准事能不能成 | 他说的话有来处——摔出来的判断力 |
| 框架合成 | 读得快拆得快整理成框架快。芒格式思维模型 | 他不是在罗列信息，他在给框架给判断 |
| AI原生 | 会用AI做工具、做内容、提效率。帮母亲做到全县第一 | 他不是讲AI概念，他是真在用 |
| 投行视角 | 资本怎么想、交易结构、定价权 | 他的判断有投行框架，不是拍脑袋 |

四层叠起来的结果：**信息散落→拼成判断。这个能力AI替代不了。** 内容池无限——每天接触新信息，每个都能拼成判断，每个判断都是一期内容。

### 信任来源

```
别人说"我懂这行" → 观众：装的吧
你展示判断力从哪来的（经历+身边案例+经典拆解+框架整理）→ 观众：这人靠得住
你来时路+实战案例 → 观众：这人的判断有来处
```

### 信任是第一 KPI

每一期内容结束后，只有一个检查标准：**观众更信任我的判断力了吗？** 不是播放量、不是点赞。

| | 汤老师新能源 | 增元AI棱镜 |
|------|------|------|
| 身份 | 能源行业专家 | **CVC投行。实战判断力+框架合成能力** |
| 讲什么 | 技术标准、政策原文 | **信息拼成框架→给出判断。不限于能源** |

## 三、内容四象限

### 四象限框架

```
                          看外部（项目/机会/行业）
                ┌──────────────────────────┐
         框架   │  行业判断               │  框架判断              │
         拆解   │  新能源/AI信号          │  投资框架/商业判断     │
                │  政策翻译/趋势拆解      │  跨领域思维模型        │
                │                          │  芒格式认知工具        │
                ├──────────────────────────┤
         故事   │  实战判断               │  来处判断              │
         经历   │  项目进展/谈判           │  来时路/身边人故事     │
                │  帮人看项目的案例        │  经典拆解→框架        │
                │                          │  读书快拆/认知分享     │
                └──────────────────────────┘
                          看内部（人/自己/认知）
```

**两条轴**：
- **横轴**：看外部（项目/机会/行业）↔ 看内部（人/自己/认知）
- **纵轴**：框架拆解（给方法给判断）↔ 故事经历（给信任给共鸣）

### 四个象限详解

| 象限 | 是什么 | 内容举例 |
|------|------|------|
| **行业判断** | 新能源/AI的信号和趋势。政策翻译。投行视角看行业 | 15分钟电价、CBAM、AI工具新进展、42号令拆解 |
| **框架判断** | 投资框架、商业底层逻辑、芒格式思维模型。不限行业 | 定价权、护城河、规模vs网络效应、反过来想、安全边际 |
| **实战判断** | 项目真实进展、谈判细节、帮人看项目的案例 | 巴南落地学到什么、朋友工厂怎么踩坑、合伙人问了什么 |
| **来处判断** | 判断力从哪来：经历、身边人案例、经典拆解、认知分享 | 四次从零开始、读芒格拆解、身边人选错的教训、三件事拼成一个判断 |

### 来处判断 · 四个来源

| 来源 | 是什么 | 观众得到什么 |
|------|------|------|
| 来时路 | 个人经历，框架怎么长出来的 | "这人真实，靠得住" |
| 身边人 | 观察到的案例，别人的选择和结果 | "他在替我看到我没看到的东西" |
| 经典拆解/读书分享 | 一本书/一个模型→快速拆→整理成框架。读得快拆得快，等于帮你读了 | "他读得快拆得快，等于帮我读了" |
| 认知分享 | 信息散落时代，你怎么收集整理形成判断 | "原来判断是这样拼出来的" |

### 阶段配比

| 阶段 | 行业判断 | 框架判断 | 实战判断 | 来处判断 | 逻辑 |
|------|---------|---------|---------|---------|------|
| **阶段1** 冷启动（前3个月） | 25% | 15% | 15% | **45%** | 先让人知道你是谁、判断力哪来的、框架怎么不一样 |
| **阶段2** 信任建立（3-6个月） | 25% | 30% | 20% | 25% | 框架判断加重，展示专业深度 |
| **阶段3** 私域转化（6个月+） | 20% | 35% | 25% | 20% | 框架+实战为主，高净值人群从这两块筛出来 |

**行业判断+框架判断永远≥40%**——专业性是底盘。来处判断在冷启动期占比最高，随信任建立自然下降。

## 四、受众分层

### 三层受众

```
公开内容 → 底层认知（所有层级通用）
  ├── 10-100万：学到方法、看到机会、建立信任
  ├── 100-500万：看到项目判断力、被筛选进圈子
  └── 500万+：看到格局和传承视角、主动来找
```

### 内容策略

- **公开内容不做人群分层**。同一期内容，不同层级各取所需。
- **高价值内容进私域/圈子**——项目机会、交易结构细节、具体判断。
- **知识产品（书/课程/工具）不主动推**，用户来问才有。
- **企业融资包装**——有融资需求的企业主找过来，微电网+AI植入→提高估值→发债/上市。投行基本功。

### 同一期内容，两种人看

```
行业判断 →
  工厂老板看到 → "这个人在帮我盯行业变化"
  同龄转行者看到 → "原来行业信号是这样拆的"

框架判断 →
  有资产的人看到 → "这个判断框架我能用"
  转行者看到 → "原来投资是这样想的"

来处判断 →
  所有人看到 → "这人的判断力是摔出来的，不是吹的"
```

## 五、内容原则

```
□ 信任是第一优先。每一期内容结束问：观众更信任我的判断力了吗？
□ AI让信息平权了。拼成框架、给出判断才是你的价值。不罗列信息，给判断
□ 来处判断是冷启动底盘——先让人知道你判断力哪来的，再展示判断力
□ 跨界视角是核心武器——实战+框架+AI+投行，四个视角叠在一起
□ 每天问自己：今天看到了什么信息→能拼成什么框架→能给出什么判断
□ 层层追问到机制层。新闻是起点不是素材，不把新闻直接丢给AI生成
□ 不懂不发。自己没搞明白的内容不出
□ 认知差筛选：讲完观众会觉得"原来是这样"还是"不过如此"？后者不发
□ 项目进展持续更新。每一步学到的东西才是内容，不是进展本身
□ 不在内容里讲道理——道理要在故事和框架里长出来
□ 不招居间人。不提居间人。内容里不出现这个词
□ 记录即资产——每次发布后提取概念/数据/判断到 Obsidian 知识库
```

## 六、日更节奏：三层金字塔

### 核心理念

日更 ≠ 追热点。日更 = 每天收集信息，每天拼框架，每天给判断。爆款是偶尔的烟花，日常是地基。

| 目标 | 需要什么 | 频率 |
|------|---------|------|
| 引流（让人看到你） | 爆款，追热点 | 偶尔，月1-2条 |
| 留人（让人记住你） | 稳定、有规律、能预期 | 每天，轻量 |
| 建信（让人信任你） | 深度、有判断、有来处 | 每周，1条深度 |

### 三层结构

```
第一层  每日学习笔记（轻量·地基·日更）
        100-200字图文卡片，5分钟写完。
        每天学到的信息拼成判断就是内容，不需要额外创作。

第二层  每周深度一条（中量·信任锚·周更）
        行业信号深拆/框架判断/项目复盘/经典拆解。
        30-60分钟，有框架有判断。

第三层  每月爆款一条（重量·引流口·月更）
        追大热点，用跨界框架给出别人给不了的判断。
        时机到了才发，不强求。
```

### 学习笔记四分类

学习笔记不限于新能源。每天接触的所有信息都该记——跨界视角是核心武器。

| 类别 | 来源 | 对应象限 | 建什么信 |
|------|------|------|------|
| 行业信号 | 储能/售电/微电网/AI的新动态 | 行业判断 | "这个人在帮我盯行业" |
| 框架方法 | 投资框架/商业模型/思维工具 | 框架判断 | "这个人有方法论" |
| 实战复盘 | 项目进展/谈判/帮人看项目 | 实战判断 | "这个人真在做项目" |
| 认知来处 | 读书拆解/身边案例/来时路 | 来处判断 | "这个人的判断有来处" |

### 周节奏

```
周一  行业判断（地基·专业）
周二  框架判断（地基·深度）
周三  来处判断（地基·信任）
周四  行业判断（地基·专业）
周五  来处判断/实战判断（地基·真实）
周六  每周深度一条（锚点）
```

一周 6 条内容，只有 1 条需要认真做，其余 5 条是学习笔记。

### 为什么学习笔记是最佳日更形态

- 你每天本来就在看信息、拼框架，记下来就行，不需要额外创作
- 100 条笔记 = 一本"判断框架词典"，未来可以做知识付费
- 观众每天看到你在看、在想、在做判断，比一次爆款建立信任更持久
- 不追热点不焦虑——热点不是你的日常，收集信息拼判断才是
- 没人在做"投行人的每日判断笔记"——这个位置是空的

### 学习笔记固定格式

```
🔋 学习笔记 | #XX

一句话判断：XX就是……

今天看到了XX → 拼上之前知道的YY → 给出一个判断：……

#学习笔记 #CVC投行 #框架思维
```

### PPT制作规范

- 工具：Canva（搜索"教育培训PPT"模板，选白底+细线+单色点缀风格）
- 比例：9:16竖版（匹配抖音全屏）
- 配色：备电暖调——琥珀[#C89050]标题色，暖深灰[#1a1e24]正文色，白底
- 字号：标题≥48pt，正文≥28pt（手机小屏可读）
- 每页一个核心信息点，不堆砌

### 录音规范

- 剪映录音，逐页念稿
- 节奏：关键概念放慢，过渡段正常语速
- 语气：像在给朋友讲一个你刚拼出来的判断，不是在上课

## 八、运营节奏

### 第1个月（冷启动）
- 日更。来处判断45%打底——先让人知道你判断力从哪来的
- 行业判断25%展示专业锚点，框架判断15%展示方法论，实战判断15%展示真项目
- 不追数据。追"有没有人觉得这人的判断不一样"
- 核心指标：有没有人说"关注了，这人看事情的角度有意思"

### 第2-3个月
- 基于数据调整四象限配比
- 来处判断可适当降低，框架判断开始加重
- 框架判断最有机会出差异化——跨界视角+芒格式思维模型，别人讲不了
- 行业判断吸引行业搜索流量，框架判断留住高净值人群

### 第3-6个月
- 项目路持续更新进展+深度拆解
- 在投行圈建立"用内容获客的CVC投资人"标签
- 实战判断+框架判断自然筛选高净值人群进私域
- 知识产品形态（用户来问才有，不主动推）

## 九、互动策略

```
□ 有人问专业问题：会的就答，不会就说"我也刚学到，我的理解是…"
□ 有人质疑：不辩解。发一期实战判断或来处判断
□ 有人问怎么合作：私信聊，不在评论区推销
□ 有人共鸣来处判断：认真回复。这些人是未来私域圈子的种子用户
□ 有人提居间人/代理：不讨论。不招居间人
```
