# 口语化脚本规则

> 解决：字太多、像念稿、没情绪。三条硬规则+改写法。

---

## 根因：为什么会"字多、念稿、没情绪"

| 症状 | 根因 | 表现 |
|------|------|------|
| 字太多 | 用书面语写作，每个信息点都想塞进去 | 一句话30字，一段话100字 |
| 像念稿 | 句子结构完整、语法正确，但没人这么说话 | "通过充电桩业务切入电力交易领域" |
| 没情绪 | 情绪写在元数据里，没写进脚本里 | 标注"情绪：好奇"，但口播全文是平的 |

---

## 规则一：硬字数限制

```
每镜 ≤ 50字（不是60，不是70）
全片 ≤ 200字（50秒以内）
```

**为什么是50字？**

50字用正常语速（5字/秒）讲就是10秒。加上自然停顿、表情、语气变化，实际占12-14秒。刚好一个镜头。

**怎么做到50字以内？**

```
砍法1：每句话只留一个信息点
  ❌ 华为最早动的，偷偷搞了好几年了，光伏逆变器已经干到全球第一
  ✅ 华为最早。逆变器全球第一。

砍法2：用短句替代长句。句号，不是逗号
  ❌ 滴滴以前匹配司机和乘客，现在匹配电厂和工厂，就是同一个算法换了个东西卖
  ✅ 滴滴以前匹配司机乘客。现在匹配电厂工厂。同一个算法。换了个东西卖。

砍法3：删填充词
  ❌ 先给你捋一下时间线啊，这个事其实特别有意思
  ✅ 给你捋一下。特别逗。

砍法4：一个镜头只讲一件事
  ❌ 镜2塞了时间线+类比+为什么（三件事）
  ✅ 镜2只讲类比，镜3讲为什么
```

## 规则二：用"呼吸单位"写脚本，不用句子

**什么是呼吸单位？**

一口气能说完的一句话。说完要换气。换气 = 句号。

```
❌ 书面语句式（朗诵稿）：
"华为数字能源业务布局多年，光伏逆变器出货量位居全球首位，
同时京东集团也通过旗下子公司获得了售电业务经营资质。"

✅ 呼吸单位（聊天）：
"华为。最早动的。
偷偷搞了好几年。逆变器全球第一。
京东。跟进。
啪。直接拿牌照。
滴滴。最近也摸过来了。"
```

**呼吸单位的三个特征：**

1. **一句不超过12个字**。超过就拆。
2. **句号不是标点，是换气点**。每个句号 = 真停顿0.3-0.5秒。
3. **一行一个呼吸单位**。换行 = 更大的停顿（0.5-1秒）。

**脚本格式规范：**

```
每行一个呼吸单位。
空行 = 换话题 = 停顿1秒以上。
重要词前面可以换行，制造停顿。

例：
以后你打开滴滴——
（停）
可能不是打车。
（停）
是买电。
（长停，让观众反应）
```

## 规则三：情绪写进脚本里，不写进元数据

**现在的问题：**

元数据写"情绪：好奇→有趣→恍然"，但口播全文是一样的节奏、一样的句式、一样的语气。情绪表是给AI看的，不是给演员看的。

**解决方案：用文字编码情绪**

| 想表达 | 脚本写法 | 效果 |
|--------|---------|------|
| 悬念 | 换行+破折号 | "以后你打开滴滴——" 自然停顿，观众等下半句 |
| 强调 | 短句+句号 | "是买电。" 两个字占一行，重量感 |
| 反转 | 单字成行 | "啪。" 一个字，制造意外感 |
| 轻松 | 口语词 | "摸过来了""特别逗""一毛一样" |
| 亲近 | 问观众 | "你能接受不？""你猜下一个是谁？" |
| 放慢 | 重复+句号 | "不用写代码。不用开公司。" 两个短句比一个长句慢一倍 |

**对比：**

```
❌ 没情绪（全是陈述句，同样长度，同样节奏）：
华为最先布局数字能源，京东随后获得售电牌照，
滴滴近期也通过小桔能源切入电力交易领域。

✅ 有情绪（长短交错，有停顿，有口语词）：
华为。最早。
偷偷搞了好几年。
京东。啪。拿牌照。
滴滴。最近也摸过来了。
```

## 改写法：从"写稿"到"对话"

### 旧流程（产出念稿）

```
1. 确定信息点：时间线+类比+为什么+画面感
2. 把信息点写成完整句子
3. 分配到镜头
4. 标注情绪
→ 产出：一篇结构完整、信息密集的"文章"
→ 问题：念起来像新闻播报
```

### 新流程（产出对话）

```
1. 确定一个核心情绪：这条视频想让观众感觉什么？
2. 用三句话讲清楚：对着手机录音，说三句就行
3. 录音转文字：这就是脚本骨架
4. 补充关键信息点：只加必要的数字、名字
5. 念一遍，删掉所有念不顺的词
→ 产出：一段你真实会说的话
→ 结果：自然、有节奏、有温度
```

### 写作时的自检清单

```
□ 念出声了吗？（没念过 = 不合格）
□ 有卡顿的地方吗？（卡 = 删掉重写）
□ 每句 ≤ 12字吗？（超过 = 拆）
□ 有连续3行以上同样长度的句子吗？（有 = 打散）
□ 有"进行""通过""基于""领域"这些词吗？（有 = 换掉）
□ 观众会在哪笑、在哪愣、在哪发弹幕？（不知道 = 没情绪）
```

## 镜头结构优化

### 不要再这样分镜

```
镜1(5s)：纯钩子，3句话，信息量为零
镜N(8s)：纯CTA，"关注我下期见"
→ 各占一个镜头但内容太薄，观众感觉在拖时间
```

### 改为这样

```
镜1：钩子直接带入第一个信息点
   "以后滴滴上能买电了。华为京东已经在干。"
   → 钩子有了，信息也有了，不浪费镜头

镜末：收尾带价值
   "这行普通人也能搞。下期讲怎么参与。"
   → 有关注理由，不是空喊"关注我"
```

### 镜头数原则

```
30-40秒 → 2-3镜
40-55秒 → 3-4镜
55-70秒 → 4-5镜

不做单镜超过15秒的（即梦限制）
不做单镜少于6秒的（内容太薄）
```

## 字数检查工具（念一遍就行）

不用算。念一遍，计时。

| 实际念完时长 | 口播字数 | 体感 | 诊断 |
|-------------|---------|------|------|
| 明显比标称短 | — | 赶、喘 | 字数太多，砍20% |
| 刚好或略超 | — | 舒服 | ✅ 过 |
| 明显超时 | — | 拖 | 信息太散，合并 |

**一条铁律：写完必须念出来。没念过 = 不合格。**

## 示例：一条脚本的改造

### 改造前（念稿感）

```
给你捋一下时间线，特别有意思。
华为最先动，数字能源做了好几年，光伏逆变器干到全球第一。
京东跟进，直接拿了售电牌照。
滴滴最近也来了，从充电桩切入做电力交易。
先搞清一个事——电网不是被抢饭碗，
电网是快递，负责把电送到你厂里。
售电公司是商家，负责卖。
互联网公司做的，是商家这层。
```
→ 114字，念起来喘不过气

### 改造后（聊天感）

```
给你捋一下。特别逗。

华为。最早。偷偷搞了好几年。逆变器全球第一。
京东。跟进。啪。拿牌照。
滴滴。最近也摸过来了。

哎，先搞清一个事。
电网不是被抢饭碗。
电网是快递。送电的。
售电公司是商家。卖电的。
互联网公司——商家这层。
```
→ 68字，有停顿、有节奏、有轻重

### 差在哪

```
改造前：每句15-25字，全是逗号连接，一口气读到底
改造后：每句2-8字，句号断开，有换行停顿

改造前：信息均匀分布，没有重点
改造后："啪""摸过来了"有轻重，节奏有起伏

改造前：书面语"切入""领域""负责把电送到你厂里"
改造后：口语"摸过来""送电的""卖电的"
```
