## 呼出流程

**Step 1**：自动加载信息——读取运营规划、客户档案、行业分析、选题库
**Step 2**：显示选题菜单——当前热点 + 系列延续 + 用户指定
**Step 3**：筛选5个选题——含标题、角度、目标人群、内容类型、**认知差评分**（★★★大部分人不知道 / ★★☆部分人不知道 / ★☆☆常识级）。认知差★☆☆的选题不发。
**Step 4**：用户选择选题
**Step 4.5**：层层追问（分析/判断/科普型强制，教程型可选）。用户抛出真实疑问 → AI 协助探索回答 → 追问到机制层（至少5层中的第3层）→ 记录完整 Q&A 链。加载 [层层追问工作流](知识库/工作流/层层追问工作流.md)。
**Step 4.6**：不懂不发闸门——用户确认三项：能用自己的话讲给外行听？被追问能再深一层？具体场景能回答？三项全✓通过。任一项✗→回 Step 4.5。反复追问仍无法通过→搁置选题。
**Step 5**：判断内容类型 → 加载对应知识库文件。将 Step 4.5 的追问链提取为内容骨架。
**Step 6**：匹配视觉风格（A 财新周刊 / B 白皮书 / C 深夜对话，按内容线自动决定）
**Step 7**：确认人设语气（五种人设选一 + 温度配比）+ 选定情绪弧（四选一）
**Step 7.5**：独家价值确认——检查素材池检索结果 + 四层独家价值三问。AI 搜不到的素材是内容差异化的根，没有独家层的内容不发。
```
三问：
1. 有没有"只有你能说"的东西？（L1 一手对话/个人经历/CVC圈子观察）
2. 观众能不能拿走一个"AI 给不了"的工具？（L2 判断框架/算账方法/避坑清单）
3. 跟 AI 直接回答差在哪一层？（L1/L2/L3/L4，至少满足一层才发）
```
**Step 8a**：导演分析（Phase 1）——先定情绪锚点（18选1，参考情绪锚点.md），再输出 directorAnalysis JSON（情绪锚点/主线/钩子/情绪曲线/观众逐页追踪/视觉比喻/关键画面）。**暂停等用户确认，禁止直接生成全文。**
**Step 8b**：分镜预设条件（仅短视频/即梦类型）——仿 zimeiti-creator 分镜前设定面板。用户确认导演分析后，依次确认以下 10 项。带 * 的为必选项，全部必选完成后才能进入 Step 8c。

|---|------|------|--------|------|------|
| 1* | 出镜形象 | 多选 | 按内容类型自动推荐 | 从 [形象库](知识库/形象库.md) 选择 | 可多选。即梦分身出镜，面部一致由分身系统保证，此处锁定服装 |

#### 时长计算公式

```
建议时长 = (口播总字数 ÷ 基准字速) × 情绪加权系数 + 停顿补偿

基准字速（语速预设）：
  慢速：5.0 字/秒（每100字≈20s）  — 深情、沉思、强调
  正常：6.7 字/秒（每100字≈15s）  — 默认
  快速：10 字/秒（每100字≈10s）   — 犀利、密集输出、紧张

情绪加权系数（情绪曲线越起伏→系数越大）：
  全程单一情绪（如纯干货）：1.0
  标准起伏（2-3个情绪转折）：1.05
  强情绪波动（4+个情绪转折，含愤怒/兴奋）：1.10

停顿补偿：
  每个空行（换话题）= 0.5s
  每个关键停顿标注（如"停一秒"）= 1s
  结尾停顿 = 1-2s（收束留白）
```

**示例计算（短视频脚本）：**

```
口播总字数：约 260 字（5镜口播内容合计）
语速：正常 → 6.7字/s → 260÷6.7 ≈ 39s 纯口播
情绪系数：标准起伏（好奇→恍然→清醒→专注→确信，5个情绪点）→ 1.05
停顿补偿：约10个空行×0.5s + 结尾停1s + 关键停顿3处×0.5s = 8s
即梦文字动画消耗：每镜约2-3s文字淡入/淡出/定格 → 5镜×2.5s ≈ 13s

建议时长 ≈ 39×1.05 + 8 + 13 ≈ 62s → 取整 60s 或 75s

若选 60s：口播语速需微提至 ~7.5字/s（偏快），情绪镜3/镜5不放慢
若选 75s：口播语速 ~5.5字/s（偏慢），金句和停顿有足够时间呼吸 → 推荐 75s
```

## 核心思维

### 封面三要素（全类型强制）

用户决定继续看一个内容，最多只花 **2 秒**。封面不是摘要，是在这 2 秒内制造认知缺口。详见 [设计规则](知识库/规则库/设计规则.md)。

```
旧：锚点 + 判断
新：粉碎默认假设 + 锚点 + 判断方向
```

五种封面模式：反常识结论式 / 身份标签式 / 数字对撞式 / 概念折叠式 / ROMO式。硬核分析默认反常识结论式。

### 背景设计铁律（全类型强制）

**严禁纯色背景。绝对禁止。** 纯白 [#ffffff]、纯黑 [#000000]、纯灰、纯任何单色——全部禁止。

每页/每镜必须有背景图，且要包含与选题相关的视觉元素（三选一，至少满足一个）：
- **渐变底**：至少两个颜色的渐变过渡（如蓝白→薄荷绿）
- **纹理底**：纸张纹理、胶片颗粒、磨砂质感、毛玻璃
- **光影底**：光束、环境光、暗角、光源方向

在此基础上，背景中必须有**至少一个与选题领域相关的抽象视觉元素**：
- 选题是储能/电池 → 流线、波形、能量网格、电路纹理
- 选题是金融/投行 → 数据流、K线纹理、几何网格
- 选题是工厂/制造 → 工业肌理、金属拉丝、蓝图线条

```
□ 每页背景是否纯色块？ → 是 → 不发。重做背景。
□ 每页背景是否有相关抽象元素？ → 没有 → 不发。
□ prompt 是否只写了颜色代码（如 #ffffff）？ → 是 → 不发。
```

**注意**：背景元素是氛围层，不是信息层。用 3-8% 透明度、虚化、半透明叠加——隐约可见即可，不能抢文字。

### 图文页面设计铁律（全类型强制）

**严禁全文字页面。** 每页必须至少有 1 个非文字视觉元素：图表、对比图、数据环、进度条、波形图、时间线、结构图——任选其一。

- 封面：数据环/进度环/大数字+图表元素，文字≤3行
- 内页：对比图/条形图/结构图/时间线 占画面 ≥40%，文字≤60字
- 禁止段落文字堆砌。能用图就不用字。

```
□ 每页是否有非文字视觉元素（图/表/环/条/线）？ → 没有 → 不发。
□ 文字是否超过 60 字？ → 超过 → 砍。
□ 这页能不能 3 秒扫完？ → 不能 → 用图代替字。
```

### GPT Image2 生图 prompt 写作铁律（A型图文强制）

**Image2 是"相机"，不是"设计师"。** 给它场景、材质、光线，它出好图。给它排版指令、抽象元素、渐变描述，它只会堆文字。

#### 核心原则：描述摄影场景，不描述图形设计

```
旧（抽象设计 — 禁止）：Soft gradient background from white to mint green. A luminous cyan ring floats center. Minimalist fintech poster aesthetic. Ample negative space.

新（摄影场景 — 必须）：Top-down shot of a dark walnut desk. Warm amber desk lamp casts a focused pool of light. A brushed brass plaque engraved with text sits in the light. The rest fades to deep shadow.
```

#### 每条 prompt 必须包含的 5 个要素

| # | 要素 | 说明 | 示例 |
|---|------|------|------|
| 1 | **物理场景** | 真实的地点/表面 | 深色木桌面、玻璃展板挂在墙上、蓝图摊在金属绘图桌上 |
| 2 | **具体材质** | 实物材料，不是抽象色块 | 黄铜铭牌、磨砂玻璃、牛皮纸、拉丝钢板、奶油色卡片 |
| 3 | **光源方向** | 光源类型+方向+色温 | 左上角窗外自然光、琥珀色台灯聚光、暖色工业吊灯 |
| 4 | **实物道具** | 场景中的真实物品 | 钢笔、计算器、咖啡杯、金属尺、文件夹、便签纸 |
| 5 | **文字载体** | 文字出现在什么实物上 | 刻在黄铜铭牌上、手写在便签纸上、印在蓝图上、标在仪表盘上 |

#### 严禁使用的抽象描述（直接拉黑）

```
❌ "gradient background"         → 改用真实材质：wood grain, brushed steel, frosted glass
❌ "clean / minimalist / airy"   → 改用具体描述：a single document on an empty desk, spot-lit
❌ "[element] floats / is placed" → 改用实物载体：printed on..., engraved on..., handwritten on a sticky note
❌ "abstract [anything]"         → 全部改成具体实物
❌ "premium / modern aesthetic"  → 改用具体材质：brass and dark wood, glass and chrome
❌ 纯色代码（#ffffff, #1a1e24）  → 改用材质描述：dark walnut wood, matte charcoal wall
```

#### prompt 质量门禁

```
□ 能不能拍成照片？ → 不能 → 重写。prompt 描述的场景必须可以用相机拍出来。
□ 文字在实物上吗？ → 浮在空中 → 重写。文字必须出现在纸张/铭牌/屏幕/卡片/玻璃等实物载体上。
□ 有光源吗？ → 没有 → 加。每条 prompt 必须至少有一个明确光源。
□ 有材质吗？ → 只有颜色代码 → 重写。必须描述物理材质（木/金属/玻璃/纸张/皮革/石材）。
□ 是不是"海报/设计稿"思维？ → 是 → 重写。切换到"摄影师拍一张照片"思维。
```

#### 页面间统一性

同一套图文的 7-8 页之间，通过以下方式保持视觉统一，同时每页又有独立的摄影场景：
- **统一光源色温**：全篇用同一种灯光色调（如琥珀暖光 或 自然白光）
- **统一材质调色板**：限定 2-3 种核心材质贯穿全篇（如深色木+黄铜+玻璃）
- **首尾呼应**：P1 和 P7 回到同一个场景/同一张桌子，形成闭环

```
□ 7页光源色温一致吗？ → 不统一 → 统一。
□ 材质调色板贯穿了吗？ → 每页完全不同材质 → 选2-3种核心材质贯穿。
□ P1和P7有呼应吗？ → 没有 → P7回到P1的场景。
```

### 开场钩子 + 超出预期（口播/视频强制）

前 1.5-3 秒决定观众是否划走。详见 [开场方法论](知识库/方法论库/开场方法论.md)。

三个万能开头公式：反常识断言式 / 身份标签式 / 静默中断式。选题类型自动匹配。

整条视频的预期管理节奏：0-3s 打破预期 → 3-15s 建立信任 → 15-45s 层层剥茧（每 15s 切换）→ 45-55s Wow Moment（75-85%位置）→ 最后 5s 收束。

Wow Moment 四种类型：认知反转 / 框架赠予 / 概念折叠 / 诚实时刻。

### 案例库驱动创作（全类型强制）

每次创作前，必须查案例库 README 索引表。案例库现有 20 篇正向案例、6 个对标账号。

**三步查法：**

```
Step A（选题阶段）：扫 README "适用场景"列 → 找到与当前选题领域/结构相近的案例
Step B（生成阶段）：读目标案例的"可复用手法矩阵" → 选 2-3 个技法嵌入当前内容
Step C（自检阶段）：用目标案例的"核心可复用点"对照当前内容 → 缺了什么？
```

**按内容类型匹配案例账号：**

| 内容类型 | 优先参考账号 | 为什么 |
|---------|------------|--------|
| A型图文（观点/判断） | 哎呀朱莉亚 | 论点宣言开场 + 证据级联 + 哲学三角收束 |
| A型图文（概念讲解） | 一二三周爸爸 | 平行世界叙事 + 隐喻翻译法 |
| 口播脚本（投行视角） | 哎呀朱莉亚 | 投行思辨表达 + 制度归因 + 跨学科引证 |
| 口播脚本（实操框架） | 爆米花企业赋能 | 三层框架 + 反例开场 + 货币化命名 |
| 清单体口播 | 投研号 #1 #2 | 清单体/深度报告体结构模板 |
| 短视频（即梦） | 一二三周爸爸 + 小德momo | 场景叙事 + 架空世界降低制作门槛 |

**反面案例的价值：**

即使案例库当前只有正向案例，创作时也要反向检查——

### 可拿走性原则

每篇内容生成后，问一句话：**观众看完能拿走什么？**

- "了解了X" → 不合格。了解 = 什么都没拿到。重写
- "能判断X" / "能算出Y" / "能避开Z" → 合格
- 答案如果只是知识 → 把知识变成工具（判断标准/计算公式/避坑清单）

反面教材：翻车案例库 5 支，病因全部指向"观众什么都没拿到"

### 七种观众心理钩子

内容必须至少触发以下一种心理，否则观众不会互动也不会看完：

| # | 心理 | 脚本策略 | 适用场景 |
|---|------|---------|---------|
| 1 | **想纠正你** | 故意留破绽/争议点，让观众忍不住评论纠正 | 知识类、教程类、盘点类 |
| 2 | **想看结果** | 开头抛悬念，用结果延迟制造好奇心 | 挑战类、实验类、对比类 |
| 3 | **想证明自己** | 抛出挑战/测试，让观众在评论区证明"我能" | 技能类、挑战类、测评类 |
| 4 | **想看你翻车** | 真实记录翻车过程，制造戏剧性和共鸣 | Vlog、教程、新手记录 |
| 5 | **想给你出招** | 求助型内容，让观众有参与感和掌控感 | 选择困难、求助、投票 |
| 6 | **想看看真假** | 揭秘/验证型，满足观众的怀疑和探究欲 | 辟谣、实测、行业揭秘 |
| 7 | **想代入自己** | 让观众看到"我也能"，建立身份共鸣 | 教程、经历分享、转型故事 |

**使用规则：**
- 每篇内容选 1 个主导心理 + 最多 1 个辅助心理，不贪多
- 选题阶段就确定心理钩子，不是生成后再贴标签
- 自检：生成后问"观众看完会做什么？"——如果答案只是"知道了"，说明没触发任何心理，重写

### 内容温度模型

三要素：有趣 + 有用 + 共鸣。至少满足两个。
情绪基调：意见领袖60% + 情绪领袖40%。

五种人设：陪伴者/教导者/崇拜者/陪衬者/搞笑者
备电定位：陪伴者（我跟你一样在学）+ 教导者（但我帮你整理好了）
