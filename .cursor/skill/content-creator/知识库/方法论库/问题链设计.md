# 问题链设计

> 一条问题链原则。7张图回答一个问题，不是7个知识点。

---

## 核心原则

每张图必须能回答"我在回答封面的哪个部分？"

---

## 问题链模板

```
P1 封面 — 制造悬念（只问不答）
P2 本质 — 到底是什么？（揭开答案）
P3 原因 — 为什么是现在/为什么是这个？（深层拆解）
P4 利好 — 有什么机会？（正面）
P5 利空 — 有什么风险？（反面）
P6 升华 — 核心看什么？（提炼一个指标）
P7 判断 — 所以呢？（收尾）
```

---

## 自检清单

```
□ 7张图能不能串成一条线？
□ 有没有哪页是孤立的知识点？
□ 随便抽掉一页，逻辑链会不会断？（会断 = 每页都必要）
□ P1 是不是只制造了悬念，没给答案？
□ P7 是不是给了判断，不是重复前面？
```

---

## 常见反模式

- 罗列知识点（"虚拟电厂的8个特征"）——换成一问一答
- 每页一个独立话题（P2讲政策、P3讲技术、P4讲市场，互不关联）——用因果链串联
- 封面给答案（"虚拟电厂：电力行业的滴滴"）——改成"滴滴和特斯拉都进来了。为什么？"
