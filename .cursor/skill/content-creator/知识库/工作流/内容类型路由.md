# 内容类型路由

> 选题→判断类型→加载对应规则。生成前必看。

---

## 决策树

```
选题来了 →
  是新闻事件/政策解读？ →
    有可交易标的（股票/赛道/概念）？ → 投资向 → 事件驱动型 → 黄金六页图文（已验证：钠电、比亚迪储能）
    无可交易标的？ → 认知向 → ⚠️ 无成熟模版，谨慎投入
  是结构性认知差（两个可交易标的对比，无事件触发）？ →
    → 投资向 → 产业对标型 → 对标六页图文（已验证：比亚迪vs宁德）
  是知识框架/产业链拆解？ → B型 1张信息图
  需要深度讲解+个人观点？ → 口播脚本（可配PPT）
  以上都不是 → 回到选题池，不硬做
```

### 投资向两个子类型

| | 事件驱动型 | 产业对标型 |
|------|----------|----------|
| 触发 | 新闻/政策/数据 | 结构性认知差 |
| 时效 | 48h窗口，过期衰减 | 无过期，可反复迭代 |
| 叙事起点 | "刚发生了一件事" | "市场一直看错了一件事" |
| 核心框架 | 事件→信号→拆解→判断 | 对标→拆解→两套标签→重估 |
| 评论驱动 | 观点认同/反对 | 身份站队（A派 vs B派） |
| 制作周期 | 快（一天） | 慢（需研究两家） |
| 方法论文档 | 黄金六页（钠电CVC拆解） | [[产业对标型图文]] |
| 成功案例 | 钠电CVC、绿证暴涨、六张网 | 比亚迪vs宁德 |
| 频率 | 每月1-2篇 | 每月1篇 |

**产业对标型的四个选题条件**：
1. 两个可交易标的（上市公司或知名品牌）
2. 同一赛道（做同一个生意，市场给不同待遇）
3. 争议空间（不能一边倒，两边都能列论据）
4. 认知标签差（不只是"A比B便宜"，是"市场为什么给A和B不同标签"）

## 类型联动

```
A型事件分析 →
  "这篇适合展开一个B型深度拆解吗？"
  "需要配一段口播吗？"
  "建议下篇选题：___"

B型知识图谱 →
  "可以配一段2分钟口播。做吗？"
  "已经有同主题的A型吗？要不要串成系列？"

口播脚本 →
  "需要配套PPT/图文吗？"
  "可以做成图文系列吗？"
```

## 加载规则

```
确定类型 →
  加载 规则库/合规规则.md + 格式规则.md + 设计规则.md
  加载 方法论库/锚点法则.md + 问题链设计.md
  加载 工作流/发布前自检清单.md
  如果是系列 → 加载 工作流/系列追踪模板.md
```
